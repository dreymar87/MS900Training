Cloud Concepts

0 / 9
Q1
A company wants to run its own customer database with full control over the operating system and storage, but doesn't want to maintain physical hardware. Which cloud service model fits this requirement?
D1
IaaS (Infrastructure as a Service) gives the customer control over the OS, storage, and installed applications, while Microsoft manages the physical datacenter and hardware. SaaS (e.g. Microsoft 365) has the provider manage everything. PaaS lets customers manage only code and data. Hybrid cloud is a deployment model, not a service model.
Q2
An organization currently stores customer data on servers in their own datacentre, but is considering moving to Azure. Which cloud deployment model describes their current setup?
D1
Their own datacentre is a private cloud — infrastructure owned and used exclusively by one organisation. Public cloud (e.g. Azure) is shared infrastructure managed by Microsoft. Hybrid cloud combines both. Community cloud is shared between specific organisations with common concerns (less commonly tested).
Q3
In which cloud service model is the customer ALWAYS responsible for managing their own data and user identities?
D1
Under the Shared Responsibility Model, the customer is responsible for data and identities in ALL service models — IaaS, PaaS, and SaaS. Microsoft manages the physical infrastructure in all three. The key variable is who manages the OS and applications, not data/identities.
Q4
A company wants to eliminate large upfront hardware costs and instead pay only for what they use each month. Which cloud benefit does this describe?
D1
CapEx (Capital Expenditure) = large upfront purchase of hardware. OpEx (Operational Expenditure) = ongoing monthly costs. Cloud's consumption-based model shifts organisations from buying hardware (CapEx) to paying for what they use (OpEx). High availability, geo-redundancy and elasticity are also cloud benefits but describe different things.
Q5
An employee works from home three days a week and from the office two days. Their IT team uses Microsoft 365 to support this setup. What work model does this describe?
D1
Hybrid work is the model where employees split time between working remotely and in the office. Microsoft 365 is specifically designed to support hybrid work — providing cloud-based apps, files, and communication tools accessible from any location or device.
Q6
Which of the following is included in Microsoft 365 but NOT in Office 365?
D1
Microsoft Intune (device management) is included in Microsoft 365 plans but not Office 365. Teams, Exchange Online, and SharePoint Online are included in both. Microsoft 365 = Office 365 + device management + advanced security. This is a classic exam distinction.
Q7
A cloud provider guarantees 99.9% uptime for their service. Approximately how much downtime per month does this allow?
D1
99.9% uptime = 0.1% downtime. Per month (720 hours): 720 × 0.001 = ~0.72 hours ≈ 43.8 minutes. This is the standard M365 SLA. Knowing this number is useful for Domain 4 (SLAs and service credits).
Q8
Contoso Ltd connects their on-premises Active Directory environment to Microsoft Azure, allowing employees to use the same credentials on-premises and in the cloud. Which cloud deployment model does this represent?
D1
Hybrid cloud connects on-premises infrastructure with a public cloud provider (here, Azure). The scenario describes hybrid identity — on-prem AD synced to Entra ID via Microsoft Entra Connect — which is a core hybrid cloud pattern. Both environments are in use simultaneously.
Q51
Fabrikam wants to move their on-premises application servers to Azure with zero code changes to the applications. Which cloud service model and migration strategy best describes this approach?
D1
IaaS lift and shift (rehost) is the migration strategy used when an organisation wants to move workloads to the cloud with zero application changes — apps simply run on Azure VMs instead of on-premises servers. SaaS repurchase means replacing on-prem software with a cloud subscription (e.g. Exchange Online instead of Exchange Server). PaaS refactor means minor code changes to leverage managed cloud runtimes. Hybrid cloud is a deployment model, not a migration strategy — it describes connecting on-premises and cloud environments.
Domain 2

M365 Apps & Services

0 / 17
Q9
A manager wants to track their team's individual tasks on a visual board, seeing which tasks are "To Do," "In Progress," and "Done." Which Microsoft 365 tool should they use?
D2
Microsoft Planner is the team Kanban-style task board — perfect for tracking work across "To Do / In Progress / Done" columns. To Do = personal tasks. Lists = structured data/tables. Loop = real-time collaborative workspaces. The Kanban board description is the key signal for Planner.
Q10
A user wants to schedule appointments with external clients who can book available time slots on a public-facing page. Which Microsoft 365 tool is designed for this?
D2
Microsoft Bookings is designed specifically for external appointment scheduling — it creates a public booking page where clients can see available slots and book time. Forms = surveys/quizzes/data collection. Outlook Calendar = internal scheduling. To Do = personal task lists.
Q11
Microsoft 365 Copilot generates content suggestions based on a user's emails, documents, and meetings. What Microsoft service provides this underlying data connection?
D2
Microsoft Graph is the unified API and data backbone that connects all M365 services — emails, calendar, files, Teams messages, and more. Copilot uses Microsoft Graph to access this organisational data (always respecting existing permissions). Entra ID handles identity. Purview handles compliance. Azure OpenAI provides the underlying LLM model.
Q12
A Teams channel stores its files in which Microsoft 365 service?
D2
Every Microsoft Teams team has a corresponding SharePoint site — channel files are stored in that site's document library. OneDrive = personal files. Stream = video recordings. Exchange = email. Teams is the front-end; SharePoint and Exchange are the backend infrastructure.
Q13
Teams meeting recordings are automatically saved to which Microsoft 365 service?
D2
Microsoft Stream is the video hosting platform in M365 — Teams meeting recordings are automatically saved to Stream (specifically, to OneDrive or SharePoint in the Stream-on-SharePoint model). Stream is for video content; it is not a document store or email system.
Q14
An IT admin wants to manage company apps on employees' personal phones without enrolling the phones into full device management. Which Intune capability achieves this?
D2
MAM (Mobile Application Management) manages apps and their data on a device without requiring full device enrolment. This is ideal for BYOD — employees' personal phones aren't controlled by IT, but company apps (Outlook, Teams) have policies applied. MDM = full device enrolment. Autopilot = zero-touch PC provisioning. Co-management = hybrid MDM with on-prem Config Manager.
Q15
A small business with 50 employees needs a simple cloud Windows desktop for remote workers. They have no IT infrastructure team and want a fixed monthly cost per user. Which solution should they choose?
D2
Windows 365 is the right choice — it's a SaaS-like Cloud PC with a fixed per-user monthly price and zero infrastructure to manage. Azure Virtual Desktop is more complex (IT must configure Azure VMs) and uses pay-as-you-go pricing — better for large enterprises wanting cost optimisation. Autopilot = zero-touch PC setup. Intune = device management, not a virtual desktop.
Q16
Which Microsoft Viva app provides employees with personal productivity analytics — such as how much time they spend in meetings versus focused work?
D2
Viva Insights provides personal wellbeing and productivity analytics — meeting hours, focus time, collaboration patterns. Connections = company intranet/news hub. Learning = learning management in Teams. Engage = social communities (formerly Yammer). Goals = OKR/objective tracking.
Q17
An admin wants to add Microsoft Power Automate workflows directly into Teams so staff can trigger approval flows from within a Teams channel. What does this represent?
D2
This describes extending Teams with collaborative apps. Power Automate, Power Apps, Planner, and Whiteboard can all be embedded into Teams as tabs or used within channels. Power Automate specifically enables automated workflows — approval flows, notifications, data processing — triggered from within Teams.
Q18
Which Microsoft 365 update channel for Microsoft 365 Apps delivers feature updates every six months and is recommended for stable, business-critical environments?
D2
Semi-Annual Enterprise Channel delivers feature updates every 6 months — ideal for regulated or business-critical environments that need stability. Current Channel = monthly feature updates. Monthly Enterprise Channel = monthly but with a predictable schedule. Insider Preview = pre-release builds for testing.
Q19
A Teams admin needs to configure call routing policies and view call quality analytics. Where should they go?
D2
The Teams Admin Center is the dedicated admin portal for Teams-specific configuration — meeting policies, call routing, Teams Phone settings, and usage analytics. The M365 Admin Center is for user/licence management. SharePoint Admin = SharePoint sites. Exchange Admin = mail flow and mailbox policies.
Q20
An employee wants to check which M365 apps they have access to and launch them from a single web page, without needing admin access. Which portal should they use?
D2
myapps.microsoft.com (or office.com) is the end-user portal — a single sign-on dashboard where any licensed user can see and launch all apps they have access to. admin.microsoft.com = admin-only management portal. Entra = identity administration. Purview = compliance management. The exam distinguishes between admin portals and the user portal.
Q21
Windows Autopilot allows a new laptop to be shipped directly to an employee who sets it up themselves on first boot. Which Intune capability works with Autopilot to automatically apply corporate policies during this setup?
D2
During Autopilot setup, the device automatically registers with Entra ID and enrols in Intune MDM, which then pushes corporate policies, apps, and settings — all without IT physically touching the device. Autopatch = automated update management. Co-management = hybrid on-prem + cloud management. Endpoint Analytics = reporting/performance data.
Q22
An IT admin wants to see how actively employees are using Microsoft Teams and SharePoint across the organisation. Where should they look?
D2
The M365 Admin Center → Reports → Usage section provides adoption analytics across all M365 workloads — Teams activity, email usage, SharePoint file activity, etc. Purview = compliance. Secure Score = security posture. Viva Insights = personal/team wellbeing analytics, not adoption reporting.
Q42
When a Microsoft Team is created, which underlying Microsoft 365 object is automatically provisioned to tie together the shared mailbox, SharePoint site, and team membership?
D2
Every Microsoft Team is backed by a Microsoft 365 Group — the object that ties together a shared inbox (Exchange), file storage (SharePoint document library), and team membership roster. Distribution Groups are email-only — no Teams workspace or SharePoint. Shared Mailboxes handle shared email access but don't provision SharePoint or a Teams workspace. Security Groups control permissions but don't automatically provision these resources.
Q43
Contoso wants to deploy a new Windows feature update to a small pilot group first, verify stability, then roll out to all employees. Which Windows-as-a-Service mechanism enables this staged approach?
D2
Deployment Rings stage Windows updates across groups: Insider/Preview → Pilot → Broad. IT can catch issues in early rings before updates reach all devices. Update Channels control update frequency for Microsoft 365 Apps (Word, Excel, etc.) — not the Windows OS. Windows Autopatch automates patch scheduling within rings, but deployment rings are the staging mechanism itself. Windows Autopilot is zero-touch device provisioning — unrelated to update staging.
Q45
Which Microsoft Viva app is designed specifically for tracking Objectives and Key Results (OKRs) and aligning individual goals to organisational priorities?
D2
Viva Goals is Microsoft's OKR platform — teams set objectives, track key results, and cascade goals upward to company-level priorities. Viva Insights covers productivity analytics — meeting hours, focus time, wellbeing. Viva Connections surfaces company intranet, news, and resources inside Teams. Viva Learning integrates LinkedIn Learning, Microsoft Learn, and internal training. Quick rule: Goals = OKRs; Insights = analytics; Connections = intranet; Learning = training; Engage = social community.
Domain 3

Security, Compliance, Privacy & Trust

0 / 16
Q23
A user is locked out of their account because they forgot their password. The IT helpdesk is unavailable. Which Microsoft Entra ID feature allows the user to resolve this themselves?
D3
SSPR (Self-Service Password Reset) lets users reset their own password using pre-registered verification methods (email, phone, authenticator app) — without contacting IT. MFA = second factor to verify identity at login. Conditional Access = policy-based access rules. Defender for Identity = detects threats in on-prem AD.
Q24
An attacker has stolen a user's Microsoft 365 password. Which feature ensures the attacker still cannot access the account?
D3
MFA (Multi-Factor Authentication) requires a second verification factor beyond the password — typically a phone app approval or SMS code. Even with the password stolen, the attacker is blocked without the second factor. SSPR = password reset. DLP = data loss prevention. Defender for Cloud Apps = SaaS app visibility.
Q25
An organisation wants to automatically require MFA when a user signs in from a new country, but allow normal sign-in from trusted office IP addresses. Which feature implements this?
D3
Conditional Access is the policy engine that evaluates context signals (location, device state, user risk) and applies conditions dynamically. "IF location is untrusted THEN require MFA" is a classic Conditional Access policy. MFA alone is all-or-nothing. SSPR is for password resets. Defender for Identity monitors on-prem AD threats.
Q26
Employees at Contoso receive convincing phishing emails containing malicious links. Which Microsoft Defender product is specifically designed to protect against this threat?
D3
Defender for Office 365 protects email and collaboration tools (Outlook, Teams, SharePoint) from phishing, malware, and malicious links — via Safe Links and Safe Attachments. Defender for Endpoint = device/endpoint protection. Defender for Identity = on-prem AD threats. Defender for Cloud Apps = SaaS/shadow IT visibility. The threat vector (email/phishing) is the key signal.
Q27
A security analyst notices that an employee's on-premises Active Directory account is being used to access resources across multiple servers in an unusual pattern. Which Defender product detects this lateral movement?
D3
Defender for Identity monitors on-premises Active Directory for anomalous behaviour — including lateral movement, pass-the-hash attacks, and compromised credential use. Key signals: "on-premises AD" + "lateral movement" = Defender for Identity. Don't confuse with Defender for Endpoint (device malware) or Defender for Cloud Apps (SaaS).
Q28
Which Zero Trust principle states that even internal users and systems should never be automatically trusted, and access decisions must always be validated?
D3
Verify Explicitly — always authenticate and authorise based on all available data points (identity, location, device, service) regardless of whether the request comes from inside or outside the network. This replaces the old "trust everything inside the firewall" model. Least Privilege = minimum necessary access. Assume Breach = design for containment assuming attackers are already inside.
Q29
An admin applies a "Confidential" label to a document that automatically encrypts the file and adds a watermark. Which Microsoft Purview feature is this?
D3
Sensitivity Labels classify and protect content — a label like "Confidential" can automatically apply encryption, add watermarks, and restrict copying or forwarding. DLP = prevents sharing of sensitive data (e.g. blocks emailing credit card numbers). Insider Risk = detects risky user behaviour patterns. eDiscovery = legal search and hold for litigation.
Q30
A customer submits a data subject request under GDPR, asking an organisation to provide all personal data held about them. Which Microsoft product helps automate the handling of this request?
D3
Microsoft Priva — Subject Rights Requests is specifically designed to manage GDPR/CCPA data subject requests — automating the discovery, review, and response to individual privacy rights requests. eDiscovery = legal/litigation hold. Insider Risk = monitoring risky employee behaviour. Purview protects organisational data; Priva protects individuals' personal data.
Q31
A security team wants a single numeric score that measures how well their Microsoft 365 environment follows security best practices, with actionable recommendations to improve it. Which tool provides this?
D3
Microsoft Secure Score is a 0–100+ numeric measurement of an organisation's M365 security posture. It lists recommended security actions with point values — completing them increases your score. Endpoint Analytics = device health/performance. Compliance Manager = regulatory compliance scoring. Viva Insights = employee wellbeing analytics.
Q32
An employee's account shows signs of being compromised — unusual sign-in location, file downloads spiking at 3am, and accessing resources never previously used. Which Defender XDR benefit helps correlate all these signals into a single incident?
D3
Defender XDR (Extended Detection and Response) correlates threat signals across identity (unusual sign-in), endpoints (file downloads), and cloud apps into a single unified incident. This cross-product correlation is the core XDR value proposition — instead of separate alerts in each product, XDR surfaces one connected attack story for analysts to investigate.
Q33
Where can IT administrators find Microsoft's published audit reports, compliance certifications (ISO, SOC), and data protection documentation?
D3
The Service Trust Portal (servicetrust.microsoft.com) is Microsoft's public transparency hub — where organisations can access third-party audit reports, compliance certifications (ISO 27001, SOC 2, FedRAMP), and data protection documentation. The Purview Compliance Portal = internal compliance management. Service Health = live incident status.
Q34
A financial services company must ensure that customer data stored in Microsoft 365 remains within the European Union for regulatory reasons. Which Microsoft feature addresses this?
D3
Microsoft Purview Data Residency ensures customer data is stored in specific geographic regions (e.g. EU-only datacentres) to meet regulatory requirements like GDPR. Sensitivity Labels protect data classification/access. Priva handles individual privacy rights. AVD geo-restriction isn't a standard M365 compliance feature.
Q41
An IT admin needs to create new user accounts and assign licenses, but should not be able to view billing invoices or change subscription plans. Which Entra ID role should be assigned?
D3
User Administrator is the least-privilege role covering user creation and license assignment — exactly what is required. Global Administrator also controls billing, security, and everything else — far more access than needed. Billing Administrator manages subscriptions and invoices, not user accounts. Security Administrator controls Defender and security policies. Key exam principle: assign the narrowest role that allows the required task (RBAC least privilege).
Q47
A compliance officer wants to track Contoso's compliance posture against GDPR and ISO 27001, with a prioritised list of improvement actions. Which tool should they use?
D3
Microsoft Purview Compliance Manager scores your organisation's posture against specific regulatory frameworks (GDPR, ISO 27001, HIPAA, etc.) and generates prioritised improvement actions. The most common distractor is Secure Score — it also scores and recommends improvements, but measures security posture (threats, attack surface), not regulatory compliance requirements. Service Trust Portal shows Microsoft's own compliance documentation. Defender Portal handles threat detection and response.
Q48
A security administrator needs to configure a Conditional Access policy that blocks sign-in attempts originating from high-risk countries. Which admin portal is used to create this policy?
D3
Microsoft Entra Admin Center (entra.microsoft.com) is where Conditional Access policies are created and managed. Conditional Access evaluates identity signals — user location, device compliance, sign-in risk, app being accessed — to grant, block, or require additional authentication. M365 Admin Center handles user and license management. Purview handles compliance, sensitivity labels, and data governance. Defender Portal handles threat detection and security incidents — not identity policy configuration.
Q49
An admin who manages only SharePoint sites should be assigned the SharePoint Admin role rather than Global Admin. Which security principle does this reflect?
D3
RBAC (Role-Based Access Control) least privilege means scoping admin permissions to exactly — and only — what the role requires. A SharePoint admin given Global Admin would gain unnecessary access to billing, security, and user management. Zero Trust is the broader security philosophy — RBAC least privilege is one way Zero Trust is implemented, but the specific mechanism described is RBAC. Conditional Access controls when and how users authenticate, not which admin roles they hold. MFA is an authentication verification method, not an access scoping mechanism.
Domain 4

Pricing, Licensing & Support

0 / 9
Q35
Contoso has 800 employees and wants a 3-year commitment with volume discounts and a single annual payment to Microsoft. Which purchasing channel should they use?
D4
Enterprise Agreement (EA) is designed for organisations with 500+ users that want volume discounts, a 3-year commitment, and consolidated annual billing directly with Microsoft. CSP = flexible monthly billing through a partner (better for SMBs). Direct = credit card purchase for small orgs. Business Premium is a plan, not a purchasing channel, and caps at 300 users.
Q36
A company purchases Microsoft 365 through a CSP partner. When they experience a Teams outage, who should they contact first for support?
D4
When purchasing through a CSP (Cloud Solution Provider), the partner is responsible for first-line support. The CSP escalates to Microsoft only if they cannot resolve the issue. This is a key CSP distinction the exam tests — CSP partners handle billing AND first-line support, not Microsoft directly.
Q37
An organisation has 250 users and needs desktop Office apps, Microsoft Intune device management, and advanced security including Microsoft Defender for Business. Which plan fits?
D4
Microsoft 365 Business Premium includes desktop Office apps (Standard adds these), plus Intune + Entra ID P1 + Defender for Business — the full security stack for SMBs. Business Basic = web/mobile apps only, no desktop Office. Business Standard = desktop apps but no Intune/Defender. E3 = Enterprise plan (250 users qualifies for Business; E3 has no user cap but costs more).
Q38
A company's Microsoft 365 service experiences an outage that breaches Microsoft's SLA. What compensation does Microsoft provide?
D4
When Microsoft breaches an SLA, organisations can claim a service credit — a billing credit applied to future invoices. This is NOT a cash refund. The credit percentage scales with the severity of the outage. This is a classic exam distinction: "service credit = billing credit, not cash."
Q39
An admin notices users cannot sign in to Microsoft 365 this morning. Where should they check first to see if there's a known Microsoft-side issue?
D4
The M365 Admin Center → Health → Service health dashboard shows real-time status of all M365 services, active incidents, advisories, and resolution timelines. For sign-in issues specifically, the Entra Admin Center also shows identity-related health. Checking service health first determines if the issue is Microsoft-side or tenant-specific.
Q40
Which of the following is an example of an add-on license rather than a base license in Microsoft 365?
D4
Microsoft 365 Copilot is an add-on license — it cannot be purchased standalone and requires a qualifying base plan (M365 E3, E5, Business Standard, or Business Premium). E3, Business Standard, and Business Premium are all base plans. Add-ons bolt additional capabilities onto an existing base plan on a per-user basis.
Q44
Microsoft Teams is completely unavailable — no user in the organisation can join meetings or send messages. An admin submits a support ticket. Which severity level should they assign?
D4
Severity A applies to complete outages with critical business impact — exactly this scenario (no user can access Teams). Sev A gets the fastest SLA response and 24/7 monitoring. Severity B is for partial impairment — some users or features affected, but the service is mostly functional. Severity C is for non-blocking issues and general questions. D is incorrect — admins assign the severity level when filing the support ticket; Microsoft does not assign it automatically.
Q46
An organisation wants the lower per-user price from a 12-month commitment, but wants to spread payments across the year rather than paying all at once. Which billing option should they choose?
D4
Annual commitment, billed monthly gives the discounted annual rate while spreading cash outflow across 12 monthly payments — the exact combination described. Monthly subscription costs more per user and requires no annual commitment. Annual billed upfront also locks in 12 months at the lower rate but requires full payment immediately. Enterprise Agreement is a separate purchasing channel for large organisations — not a billing frequency choice for standard subscriptions.
Q50
Teams is experiencing intermittent audio issues for some users during meetings, but most Teams functions are still working normally. Which support severity is most appropriate for the support ticket?
D4
Severity B applies to partial impairment — some users or features affected, but the service isn't completely down. Intermittent audio for some users while Teams otherwise works = Sev B. Severity A requires a complete outage affecting all users with critical business impact. Severity C is for minor non-blocking issues or general questions. D is wrong — an actively impaired service warrants a support ticket; the Service Health dashboard is for checking status, not a substitute for filing.
