⚡ TL;DR — What You Must Know
  • Three buying channels: Enterprise Agreement (EA) for large orgs · Cloud Solution Provider (CSP) for SMBs · Direct billing from Microsoft
  • Business plans (up to 300 users): Basic, Standard, Premium. Enterprise plans (unlimited): E3 and E5
  • Base license = the core M365 plan. Add-on license = extras bolted on (e.g. Copilot, Teams Phone, Defender P2)
  • Billing: monthly or annual commitment; per-user per-month pricing model
  • Support: admin submits ticket in Admin Center → Microsoft (or CSP) responds. Severity determines response time
  • SLA: Microsoft guarantees 99.9% uptime for most M365 services. Downtime can earn service credits
  • Service health: check real-time status at admin.microsoft.com or Entra Admin Center

The Three Purchasing Channels

Organizations can buy Microsoft 365 licenses through three routes. The exam tests your ability to match the right model to an org's size and needs.

ChannelBest ForKey Traits
Enterprise Agreement (EA)500+ users / large orgs3-year commitment; volume discounts; annual upfront payment; predictable costs
Cloud Solution Provider (CSP)SMBs (any size, min 1 user)Monthly billing through a Microsoft partner; flexible; partner provides support
Direct / Microsoft.comSmall orgs / individualsBuy directly; credit card; monthly or annual; self-support via documentation
📌
Key CSP distinctionWhen buying through a CSP partner, the partner handles billing AND first-level support — not Microsoft directly. The CSP escalates to Microsoft only if they can't resolve the issue.
M365 License Plans — Business vs Enterprise

Microsoft 365 comes in plan families. The most important distinction is Business (≤300 users) vs. Enterprise (unlimited users).

PlanUser LimitKey Inclusions
M365 Business Basic≤300Web/mobile Office apps, Teams, Exchange, SharePoint, OneDrive
M365 Business Standard≤300Above + desktop Office apps
M365 Business Premium≤300Above + Intune + Entra ID P1 + Defender for Business
M365 E3UnlimitedDesktop apps + advanced compliance + Intune + Entra ID P1
M365 E5UnlimitedE3 + advanced security (Defender P2, Purview) + Power BI Pro + Teams Phone
💡
Exam scenario pattern"An org has 400 users and needs advanced threat protection" → Must be Enterprise (E3 or E5) because Business plans cap at 300.
Base Licensing vs Add-On Licensing

M365 uses a modular licensing model. Every user needs a base plan; additional capabilities can be added through add-on licenses assigned to specific users.

Base LicenseCore M365 plan (e.g. M365 E3 — the foundation)
Add-On LicenseExtra features bolted on top (per user)
Common Add-OnsWhat It Adds
Microsoft 365 CopilotAI assistant in all M365 apps (requires E3/E5 base)
Teams PhonePSTN calling capability in Teams
Microsoft Defender P2Advanced identity protection (if not already in plan)
Power BI ProFull Power BI sharing and collaboration
Billing Options & License Management

Microsoft 365 subscriptions are priced per-user, per-month. Admins manage all billing and license assignments from the Microsoft 365 Admin Center.

Monthly billingFlexible, slightly higher per-unit cost
Annual commitmentLower cost; billed monthly or upfront annually
License assignmentAdmin assigns/removes licenses per user in Admin Center
License reclamationWhen users leave, licenses can be reassigned

How to Get Support for M365

Support for M365 is tiered. The primary channel for admins is the Microsoft 365 Admin Center, where support tickets (called "service requests") are submitted with a severity level.

📞 Microsoft Support
Admin opens ticket in Admin Center. Phone support available. Available 24/7 for critical issues.
Direct
🤝 CSP Partner Support
If purchased via CSP, the partner is first-line support. They escalate to Microsoft if needed.
Partner-led
🌐 Community & Docs
Microsoft Tech Community, Q&A forums, Microsoft Learn documentation for self-service help.
Self-service

How to file a support request — step by step:

StepAction
1. Open Admin CenterGo to admin.microsoft.com → Sign in as admin
2. Navigate to SupportLeft nav → Support → New service request
3. Describe the issueType a description — virtual agent suggests self-help first
4. Escalate to agentSelect "Contact support" for phone or email with a Microsoft engineer
5. Assign severitySev A (critical, service down) · Sev B (impaired) · Sev C (general question)

Microsoft support plan tiers — what's included at each level:

PlanWho It's ForKey Features
Basic / SubscriptionIncluded with all M365 plansSelf-help resources, community forums, billing support, submit tickets for break-fix issues
DeveloperDev/test tenantsTechnical support for development scenarios during business hours
StandardSMB / growing orgs24/7 break-fix support, 1-hour critical response SLA, designated support contacts
Professional DirectBusiness-critical workloadsProactive guidance, advisory services, faster response times, dedicated account team
Unified (Premier)EnterpriseDeepest integration, designated Technical Account Manager (TAM), on-site support available
📌
Exam tip — three things to know cold 1. Support requests are filed in the M365 Admin Center → Support (not the Defender or Purview portals). 2. Severity A = critical business impact, 24/7 response. Severity C = general question, next business day. 3. CSP customers contact their partner first, not Microsoft — the partner is first-line support.
Service Level Agreements (SLAs) & Service Credits

Microsoft publishes SLAs that guarantee uptime for each M365 service. If Microsoft fails to meet the SLA, organizations can claim service credits as compensation — applied to future billing.

Uptime GuaranteedMeaningMax Downtime / Month
99.9%Standard SLA for most M365 services~43.8 minutes/month
<99.9%SLA breach → eligible for service creditsService credit % scales with outage severity
Service CreditBilling credit issued when SLA is breached — NOT a cash refund
Financially-backed SLAMicrosoft's formal contractual uptime commitment
Checking Service Health Status

Admins can check the real-time health of all M365 services through two portals. This lets you quickly determine if an issue is on Microsoft's side or within your tenant.

PortalPath to Service HealthWhat You See
M365 Admin Centeradmin.microsoft.com → Health → Service healthCurrent incidents, advisories, affected services, resolution timeline
Entra Admin Centerentra.microsoft.com → Service healthIdentity-specific health (Entra ID authentication issues)
💡
Message CenterAlso in the Admin Center, the Message Center shows planned changes, new features, and maintenance notifications — separate from incident-based service health alerts.
