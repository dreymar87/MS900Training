Core Microsoft 365 Apps

The foundation of M365 productivity. These are fully cloud-connected, always up-to-date SaaS applications — no manual patching.

AppPrimary PurposeKey Capability

Microsoft 365 Copilot & Copilot Chat

Copilot is Microsoft's AI layer embedded directly into M365 apps. It uses your organizational data (emails, documents, meetings) via the Microsoft Graph to generate context-aware assistance. Copilot Chat is a web-grounded, enterprise-safe AI chat experience.

Copilot in WordDrafts, rewrites, summarizes docs
Copilot in TeamsMeeting summaries, action items
Copilot in OutlookEmail drafts, thread summaries
Copilot ChatEnterprise-safe AI chat with web grounding
Microsoft GraphThe data backbone Copilot uses
📌
Exam tipCopilot is scoped to data the user already has permission to access — it doesn't bypass existing M365 permissions. This is a key distinction for security questions.

Microsoft Teams — The Collaboration Hub

Teams is the central collaboration platform in M365. It integrates chat, video meetings, calling, file storage (via SharePoint), and extensibility through apps and connectors.

Teams ChatPersistent 1:1 and group messaging
Teams MeetingsVideo/audio with recording, transcripts
Teams PhonePSTN calling from Teams (replaces PBX)
Teams ChannelsThreaded conversations per topic/project
Teams AppsPlanner, Power Apps, Whiteboard integrations
💡
Key relationshipEvery Team has a corresponding SharePoint site (for file storage) and an Exchange mailbox (for meeting invites). Teams is the front-end; SharePoint and Exchange are the backend.
SharePoint, OneDrive & Stream

These three services handle content storage and sharing at different scopes.

ServiceScopePrimary Use
SharePointOrganizationalIntranet sites, document libraries, team collaboration, workflows
OneDrivePersonalIndividual cloud file storage; sync to desktop; share with others
StreamOrganizationalVideo hosting, recording storage (Teams recordings go here)
⚠️
Don't confuse theseSharePoint = team/org files (shared). OneDrive = MY files (personal, but shareable). Every Teams channel stores its files in a SharePoint library — OneDrive is not used for Teams files. Stream is video-only — Teams meeting recordings automatically go to Stream.
Exchange Online & Outlook

Exchange Online is the cloud-hosted email and calendaring service. Outlook is the client (app) used to access it — available as desktop, web (OWA), and mobile. Exchange powers mailboxes, shared calendars, distribution groups, and resource booking (rooms, equipment).

Exchange OnlineBackend email service (cloud)
OutlookClient app to access email & calendar
OWAOutlook Web App — browser-based
Distribution GroupEmail alias that routes to multiple users
Extending Teams with Collaborative Apps

Teams is an extensible platform — organisations add purpose-built apps directly into Teams channels, meetings, and chats. The exam tests your ability to identify what each extension does.

AppWhat It Adds to TeamsPrimary Use
WhiteboardInfinite digital canvas in meetingsReal-time visual brainstorming and ideation during calls
PlannerKanban task boards as a Teams tabTrack team tasks without leaving Teams
Power AppsCustom no-code/low-code business apps embedded in Teams tabsBuild tailored tools (e.g. leave requests, inventory forms) without dev resources

💡
Copilot in collaborationMicrosoft 365 Copilot also extends Teams specifically for collaboration — it summarises missed meetings, generates recap notes with action items, and can answer questions about channel content. This is the collaboration-specific Copilot capability (distinct from its productivity role in Word/Excel).

Microsoft Intune — MDM & MAM

Intune is the cloud-based Mobile Device Management (MDM) and Mobile Application Management (MAM) solution in M365. It lets IT enforce policies on devices (Windows, iOS, Android, macOS) and manage which apps can access company data — even on personal (BYOD) devices.

CapabilityWhat It Does
MDM (Device enrollment)Full device management — enforce encryption, compliance policies, remote wipe
MAM (App protection)Manage apps on personal devices without enrolling the device itself (BYOD-friendly)
Co-managementHybrid: manage devices with both Intune and on-prem Configuration Manager simultaneously
Endpoint AnalyticsReports on device health, startup performance, app reliability
Windows AutopatchAutomates Windows and Office update management across device fleets
Windows Autopilot — Zero-Touch Provisioning

Windows Autopilot allows organizations to ship new devices directly to employees who self-provision them with corporate settings — no IT imaging or manual setup required. The device registers in Entra ID and pulls Intune policies automatically on first boot.

💡
AnalogyAutopilot is like ordering a company laptop that "configures itself" when the employee first turns it on. IT never physically touches the device.
Windows 365 vs Azure Virtual Desktop

Both deliver cloud-based Windows desktops, but with different models. This is a common comparison question on the exam.

Windows 365Azure Virtual Desktop (AVD)
ModelFixed per-user Cloud PC (SaaS-like)Flexible multi-session virtualization (IaaS-like)
ComplexitySimple — no infrastructure to manageComplex — IT controls VM sizing, sessions, networking
PricingFixed monthly per-user licensePay-as-you-go (compute & storage)
Best forSMBs, predictable workloadsEnterprises needing cost optimization at scale
Multi-sessionNo (dedicated per user)Yes (multiple users per VM)
⚠️
The exam trapBoth give you a cloud Windows desktop — the difference is who manages the infrastructure. Windows 365 = Microsoft manages everything, fixed price per user, simple. AVD = IT manages the Azure infrastructure, pay-as-you-go, complex but cheaper at scale. Scenario tip: "small org, easy setup" → Windows 365. "large enterprise, cost optimisation, multi-session" → AVD.
Windows-as-a-Service (WaaS) & Update Channels

WaaS is Microsoft's continuous delivery model for Windows — no more big version upgrades every few years. Feature updates are delivered regularly, and organizations control rollout using deployment rings.

RingTarget AudienceUpdate Timing
Insider / PreviewIT volunteers, early adoptersEarliest — pre-release builds
PilotRepresentative sample of production usersAfter Insider validation — limited production rollout
BroadAll remaining devices / general workforceLast — fully validated, org-wide deployment
📌
Exam tip — rings vs channelsDeployment rings (Insider → Pilot → Broad) control when Windows updates reach devices. Update channels (Current, Monthly Enterprise, Semi-Annual) control how often Microsoft 365 Apps receive feature updates. These are two different mechanisms — don't mix them up.

M365 Administration Capabilities

Microsoft 365 Admin Center

The M365 Admin Center (admin.microsoft.com) is the central management hub for global admins. From here, admins manage users, licenses, billing, service health, and reports — and navigate to service-specific admin centers.

User managementCreate, edit, delete users; assign licenses
Usage reportsAdoption data: Teams, Email, SharePoint activity
Service healthView current M365 service incidents & advisories
Message centerUpcoming changes & feature announcements
Service-Specific Admin Centers & Copilot Dashboard

Each major M365 workload has its own admin center with deeper controls and reports.

Admin CenterKey Reports / Controls
Teams Admin CenterMeeting policies, call routing, Teams usage analytics, app permissions
SharePoint Admin CenterSite management, sharing settings, access controls, storage quotas
Exchange Admin CenterMail flow rules, mailbox policies, anti-spam, message trace
Copilot DashboardAI usage insights — how Copilot is being used across the org, adoption metrics
📌
Exam tipThe Copilot Dashboard is accessed from the M365 Admin Center and helps admins monitor AI adoption — this is a newer exam addition, don't overlook it.
The Microsoft 365 User Portal

Distinct from the Admin Center (which is admin-only), the M365 user portal is the self-service experience for end users. Accessible at office.com or myapps.microsoft.com, it lets any licensed user manage their own account and access their apps — no admin rights required.

PortalWho Uses ItWhat They Can Do
office.comAll licensed usersLaunch all M365 apps (Teams, Outlook, Word, SharePoint...), access OneDrive files, install Office desktop apps
myaccount.microsoft.comAll licensed usersManage security info, update MFA methods, view/manage devices, download Office, check assigned licenses
myapps.microsoft.comAll licensed usersSingle sign-on dashboard — see and launch all apps the user has been granted access to
📌
Exam tip — Admin Center vs User PortalAdmin Center (admin.microsoft.com) = IT admins only; manage the whole tenant. User portal (office.com / myapps) = individual employees; manage their own experience. The exam will test this distinction in scenario questions.
