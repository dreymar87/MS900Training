What is Cloud Computing?

Cloud computing is the delivery of computing services — including servers, storage, databases, networking, software, and analytics — over the internet. Instead of buying and maintaining physical data centers, you rent capacity from a cloud provider and pay only for what you use (OpEx model vs. traditional upfront CapEx).

CapExUpfront hardware purchase
OpExPay-as-you-go, ongoing
Consumption-basedBilled only on usage
Shared Responsibility Model

In cloud, security responsibilities are split between Microsoft (the provider) and you (the customer). What you're responsible for depends on the service model — the higher up the stack, the less you manage.

ResponsibilityOn-PremisesIaaSPaaSSaaS
Physical datacenterCustomerMicrosoftMicrosoftMicrosoft
OS / Virtual MachinesCustomerCustomerMicrosoftMicrosoft
ApplicationsCustomerCustomerCustomerMicrosoft
Data & IdentitiesCustomerCustomerCustomerCustomer
📌
Exam tipData, identities, and endpoints are ALWAYS the customer's responsibility regardless of service model — this is a common exam question.
⚠️
Don't mix up service model vs deployment model These are two different classifications — easy to conflate. Service model (IaaS/PaaS/SaaS) = what is managed for you. Deployment model (Public/Private/Hybrid) = where the infrastructure lives and who can access it. M365 is both a SaaS service model and delivered from the public cloud deployment model — these answers are independent.
Public, Private & Hybrid Cloud Models

Cloud deployment comes in three flavors. Microsoft Azure is a public cloud. Many enterprises run hybrid to connect their on-premises datacenters to Azure services.

IaaS, PaaS & SaaS — The Service Models

These three models represent increasing levels of abstraction. Think of it as renting vs. leasing vs. buying a car.

ModelWhat You GetYou ManageMicrosoft Example
IaaSVMs, networking, storageOS, middleware, apps, dataAzure Virtual Machines
PaaSManaged runtime/platformApp code & data onlyAzure App Service
SaaSFully managed applicationJust your data & usersMicrosoft 365, Teams, Outlook
💡
The MS-900 angleMicrosoft 365 is a SaaS product. You don't manage servers, OS patches, or app updates — Microsoft handles all of that. Your job is managing users, data, and policies.
⚠️
Don't confuse the three models The exam will give you a scenario and ask which model fits. The key question is: who manages the OS? — IaaS: you manage OS + everything above it. PaaS: Microsoft manages OS and runtime; you manage only your code and data. SaaS: Microsoft manages everything — you just use the app. Quick check: "needs to manage their own OS" → IaaS. "Just wants to deploy code" → PaaS. "Just wants to use an app" → SaaS (M365 is SaaS).

Benefits of Cloud & Hybrid Work

Know these cloud benefits — they're frequently tested in scenario questions asking "why would an organization move to the cloud?"

High AvailabilityUptime guarantees via SLAs (e.g. 99.9%)
ScalabilityScale up/out on demand
ElasticityAuto-scale up AND down
ReliabilityFault tolerance, geo-redundancy
AgilityProvision resources in minutes
CapEx → OpExTrade big upfront spend for monthly billing

Hybrid work refers to the model where employees split time between office and remote. Cloud-based M365 makes this seamless — apps, files, meetings, and devices are all managed centrally and accessible from anywhere.
