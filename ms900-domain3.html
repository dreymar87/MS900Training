⚡ TL;DR — What You Must Know

Microsoft Entra ID — Cloud Identity Platform

Microsoft Entra ID (formerly Azure Active Directory / Azure AD) is Microsoft's cloud-based identity and access management service. Every M365 tenant uses Entra ID as its identity backbone. It authenticates users and authorizes access to M365 apps, SaaS apps, and on-premises resources.

AuthenticationVerifying WHO you are (login)
AuthorizationVerifying WHAT you can access (permissions)
SSOSign in once, access all apps (Single Sign-On)
TenantYour org's dedicated Entra ID instance
📌
RBAC — Role-Based Access ControlEntra ID uses RBAC to limit what admins can do — assign only the least-privileged role needed. Key roles to know: Global Admin (full M365 control — use sparingly) · User Admin (create/edit users and assign licenses) · Billing Admin (manage subscriptions and payments) · Security Admin (configure Defender and Secure Score). Exam tip: a scenario asking "who should manage user accounts without touching billing?" → User Admin, not Global Admin.
Cloud, On-Premises & Hybrid Identity

Organizations can manage identities in three ways depending on their infrastructure.

Identity TypeWhere ManagedHow It Works
Cloud-onlyEntra ID (cloud)Users exist only in the cloud — no on-prem AD
On-premisesActive Directory (on-prem)Traditional AD DS; not cloud-connected
HybridBoth (synced)On-prem AD synced to Entra ID via Entra Connect; one identity for both worlds
📌
Exam tipMost enterprises use Hybrid identity — employees have one AD account that works on-prem AND in the cloud. Entra ID Connect is the sync tool that bridges them.
Microsoft Entra ID Identity Types — Users, Devices, External & Workload

Entra ID manages four categories of identities. The exam tests your ability to pick the right identity type for a given scenario — especially around BYOD, B2B collaboration, and non-human service accounts.

👤 User Identities

Member (Internal)Full-time employees and staff — internal Entra ID accounts
Guest (External)B2B collaborators — bring their own credentials from another org or IdP

💻 Device Identity Types

Device TypeDescriptionUse Case
Microsoft Entra Registered ⭐Personal device registered with the organization. User signs in with their personal account. IT can apply basic policies (e.g., require PIN, block jailbroken devices) without full MDM enrollment.BYOD (Bring Your Own Device) — employees using personal phones or laptops to access work apps
Microsoft Entra JoinedOrganization-owned device fully joined to Entra ID (cloud only, no on-prem AD). Managed by Intune. Best for cloud-first environments.New corporate-issued Windows devices in cloud-only or hybrid organizations
Hybrid Entra JoinedDevice is joined to both on-premises Active Directory AND Microsoft Entra ID simultaneously. Synced via Entra Connect/Cloud Sync.Existing corporate devices in hybrid environments with on-prem AD infrastructure
📌
Exam tip — device type scenarios "Employee uses their personal laptop to access work apps (BYOD)" → Microsoft Entra Registered
"New org-owned device, cloud-only setup" → Microsoft Entra Joined
"Existing org devices with on-premises AD that also need cloud access" → Hybrid Entra Joined

🌐 External Identities (B2B)

External identities allow partners, vendors, and consultants to access your organization's resources using their own credentials from their home organization or identity provider (Google, Microsoft, any SAML/OIDC provider). No need to create a new account — they authenticate with what they already have.

📌
Exam tip — external users & B2B"Consultant needs temporary access to internal resources" → External guest identity. "Org wants secure access for external users without issuing new credentials" → Allow external users to authenticate with their own identity providers (External Identities / B2B). B2B collaboration = External Identities.

⚙️ Workload Identities

Non-human identities assigned to applications, services, scripts, and automated processes that need to authenticate and access resources. Examples: a web app that reads SharePoint data, a script that updates Azure resources, a CI/CD pipeline that deploys code.

📌
Exam tip — workload vs device identities"A printer in the office network needs an identity" → Device identity. "An app or service needs to authenticate to access M365 data" → Workload identity. Neither is a user identity.

🔗 Microsoft Entra Cloud Sync

Microsoft Entra Cloud Sync (previously Azure AD Connect) is the tool that synchronizes on-premises Active Directory identities to Microsoft Entra ID in the cloud. It enables hybrid identity — one set of user credentials works for both on-prem and cloud resources. Essential for organizations transitioning to the cloud while maintaining on-prem infrastructure.

📌
Exam tip — hybrid identity integration"Which component is essential for seamless integration between on-premises Active Directory and Microsoft Entra ID?" → Microsoft Entra Cloud Sync. Also: "Microsoft Entra ID acts as a single identity provider for both on-premises and cloud environments in a hybrid setup" — this is the correct description of its role in hybrid scenarios.
MFA, SSPR & Conditional Access

These three features work together to secure identities without creating excessive friction for users.

FeatureWhat It DoesWhy It Matters
MFA
(Multi-Factor Auth)		Requires 2+ verification factors: password + phone app / SMS / biometricsEven if password is stolen, attacker can't get in without the second factor
SSPR
(Self-Service Password Reset)		Users reset their own passwords using verified methods (email, phone, security questions)Reduces helpdesk tickets; users aren't locked out
Conditional AccessPolicy engine: "IF user is signing in from an unknown location, THEN require MFA"Contextual, risk-based access — tighter security with less friction for trusted scenarios
💡
The signals Conditional Access usesUser/group identity · Device compliance state · Location (IP/country) · Client app type · Sign-in risk level (from Entra ID Protection)
⚠️
Scenario trap — pick the right tool "User forgot their password and is locked out" → SSPR (self-service reset, no helpdesk needed). "Attacker has the user's password but can't get in" → MFA (second factor blocks them). "Block access from high-risk countries but allow trusted office IPs" → Conditional Access (context-based policy). The exam describes a scenario — know which tool solves which problem.

Microsoft Defender XDR — Unified Threat Protection

Defender XDR (Extended Detection and Response) is Microsoft's integrated security suite that correlates threat signals across all attack surfaces — endpoints, email, identity, and cloud apps — into a single view in the Microsoft Defender Portal.

Defender ProductProtectsKey Capability
Defender for EndpointDevices (Windows, macOS, Linux, mobile)Threat detection, EDR, vulnerability management
Defender for Office 365Email, Teams, SharePoint, OneDrive — malicious links, attachments, and collaboration threatsAnti-phishing, Safe Links (scans URLs), Safe Attachments (sandboxes files before delivery)
Defender for IdentityOn-premises Active Directory identitiesCloud-based solution that identifies, detects, and investigates advanced threats, compromised identities, and malicious insider actions directed at your organization via on-prem AD monitoring
Defender for Cloud AppsSaaS applications and cloud services (including Shadow IT)Cloud Access Security Broker (CASB) — discovers unsanctioned apps, enforces session controls, identifies sensitive data in cloud apps
Defender Vulnerability ManagementDevices and software across the organizationIdentifies and helps remediate critical vulnerabilities and misconfigurations — provides asset inventory, vulnerability assessment, and prioritized remediation guidance

Defender for Cloud Apps — CASB Framework & 4 Pillars

Defender for Cloud Apps is a Cloud Access Security Broker (CASB). A CASB sits between cloud service users and cloud providers to enforce security policies. The Cloud App Security framework is built on four pillars:

PillarWhat It Does
VisibilityDetect all cloud services in use — including unauthorized Shadow IT apps that IT doesn't know about
Data Security ⭐Identify and control sensitive information in cloud apps — prevent sensitive data from being shared or leaked through unsanctioned services
Threat ProtectionDetect and remediate cyberthreats across cloud services — anomalous activity, compromised accounts, malicious insiders
ComplianceAssess the compliance posture of cloud services against regulations (GDPR, HIPAA, ISO 27001) and restrict non-compliant apps
💡
The XDR advantageInstead of each product working in isolation, Defender XDR correlates signals across all four pillars. An attack that starts with a phishing email → credential compromise → lateral movement is detected as one unified incident.
⚠️
Don't confuse these Defender products for Endpoint = Windows/Mac/Linux devices. for Office 365 = email & collaboration (phishing, malware in attachments). for Identity = on-premises Active Directory accounts (NOT cloud users). for Cloud Apps = SaaS apps and shadow IT (CASB). Vulnerability Management = finding and fixing vulnerabilities and misconfigurations on devices. Memory trick: the name tells you what it protects.
📌
Key exam answers to lock in "Protect against malicious threats posed by email messages, links (URLs), and collaboration tools" → Microsoft Defender for Office 365
"Identifies, detects, and investigates advanced threats, compromised identities, and malicious insider actions" → Microsoft Defender for Identity
"CASB pillar that identifies and controls sensitive information" → Data Security
"Address critical vulnerabilities and misconfigurations across the organization" → Microsoft Defender Vulnerability Management
"Quickly understand the organization's current security posture in the Defender portal" → Secure Score
🖥️
The Microsoft Defender PortalAll four Defender products are managed from a single unified admin interface: security.microsoft.com — the Microsoft Defender Portal. Admins investigate incidents, review alerts, manage secure score, and run threat hunting from here. Separate from the Purview compliance portal (compliance.microsoft.com) and the M365 Admin Center (admin.microsoft.com).
Common Threats & How M365 Addresses Them

The exam tests your ability to map real-world attack types to the M365 tools that defend against them. Think of this as "if the threat is X, the answer is Y."

Threat TypeAttack VectorM365 Defence
Phishing / malicious emailFake emails trick users into clicking malicious links or attachmentsDefender for Office 365 — Safe Links, Safe Attachments, anti-phishing policies
Credential theft / password attacksBrute force, credential stuffing, or stolen passwordsMFA + Conditional Access + SSPR — second factor blocks access even with stolen passwords
Malware / ransomware on endpointsMalicious software installed on Windows/macOS devicesDefender for Endpoint — EDR, behavioural analysis, automated remediation
Compromised on-premises identityAttacker steals AD credentials and moves laterallyDefender for Identity — monitors on-prem AD for anomalous behaviour and lateral movement
Unsanctioned cloud apps (Shadow IT)Employees use unapproved SaaS apps that IT can't see or controlDefender for Cloud Apps — discovers shadow IT, enforces session controls and policies
Insider threats / data exfiltrationEmployees intentionally or accidentally leak sensitive dataPurview Insider Risk Management + DLP — detects and prevents data leaks
📌
Exam tipScenario questions often describe a threat and ask "which M365 tool should be used?" Learn the threat → tool mapping above cold. The most common traps: Defender for Identity = on-prem AD threats (not cloud); Defender for Cloud Apps = shadow IT/SaaS visibility (not endpoint protection).

Secure Score is a numeric measurement (0–100+) of your organization's security posture in Microsoft 365. A higher score means more recommended security controls are in place. It provides actionable recommendations with estimated score improvements.

Zero Trust Model

Zero Trust is a security philosophy that replaces the old "castle and moat" model. Rather than trusting everything inside your network, Zero Trust assumes breach and verifies every request explicitly.

🔍
Verify Explicitly
Always authenticate using all available data points (identity, location, device, service, workload)
🔒
Least Privilege
Limit user access with just-in-time (JIT) and just-enough-access (JEA)
🛡️
Assume Breach
Minimize blast radius. Segment access. Encrypt. Use analytics to detect threats.
📌
Zero Trust → M365 feature mapping The exam tests whether you can match Zero Trust principles to the specific M365 tools that implement them:

🔍 Verify Explicitly → MFA (verifies identity with multiple factors) + Conditional Access (evaluates every sign-in using identity, device, location, and risk signals)
🔒 Least Privilege Access → RBAC (admin roles scoped to minimum permissions) + Entra ID Privileged Identity Management (just-in-time access)
🛡️ Assume Breach → Defender XDR (detects breaches across all surfaces) + Purview eDiscovery & Auditing (investigate when breach occurs) + Sensitivity Labels (data protected even if stolen)

Memory hook: MFA = Verify. RBAC = Least Privilege. Defender = Assume Breach.
Microsoft Purview — Compliance Suite

Microsoft Purview is the umbrella brand for Microsoft's information protection and compliance tools. It helps organizations protect sensitive data, manage risk, and meet regulatory requirements.

Purview FeatureWhat It Does
Sensitivity LabelsClassify and protect documents/emails (e.g. "Confidential" label enforces encryption automatically)
Data Loss Prevention (DLP)Detects and prevents sharing of sensitive data (e.g. blocks emailing credit card numbers externally)
Insider Risk ManagementDetects risky user behaviors like data exfiltration or policy violations from within the org
eDiscoveryLegal hold and search across emails, Teams chats, SharePoint for litigation or investigation
AuditingUnified audit log of user and admin activities across M365 services
Data ResidencyEnsures customer data is stored in specific geographic regions for regulatory compliance
Compliance ManagerDashboard that scores your compliance posture against regulations (GDPR, ISO 27001, NIST) and provides improvement actions
📌
Compliance Manager vs Secure ScoreThese are two separate scores — easy to confuse. Secure Score (in the Defender Portal) measures your security posture. Compliance Manager (in the Purview portal) measures your regulatory compliance posture against specific frameworks like GDPR or ISO 27001. Both give you a score and a list of improvement actions, but they answer different questions.
Purview eDiscovery, Audit, & Communication Compliance — Detailed

Three Purview capabilities that appear frequently in exam questions with specific feature-level details you need to know.

🔍 eDiscovery — Workflow & Key Feature

eDiscovery allows organizations to search, preserve, and export electronically stored information (ESI) for legal cases or internal investigations. The typical workflow:

StepAction
1. Identify trigger eventA legal case, regulatory inquiry, or internal investigation begins
2. Create and manage caseSet up the eDiscovery case in the Microsoft Purview portal to organize all related activities
3. Search and evaluate resultsRun content searches across emails, Teams chats, SharePoint, OneDrive; refine queries
4. Create Holds ⭐Preserve content relevant to the case to prevent deletion — a Hold places a lock on mailboxes or sites so content cannot be deleted or modified during the investigation
5. Export search resultsDownload collected evidence for legal review or submission
📌
Exam tip — eDiscovery Holds"During an investigation, you need to preserve content relevant to an eDiscovery case to prevent deletion" → Create Holds. Also: "Can't access eDiscovery features in the Purview portal?" → Verify that appropriate permissions are assigned (eDiscovery Manager or eDiscovery Administrator role required).

📋 Audit (Standard) vs Audit (Premium)

Microsoft Purview Audit captures user and admin activity across M365 services. Two tiers with important differences:

FeatureAudit (Standard)Audit (Premium)
Log Retention90 daysUp to 1 year (and optionally 10 years with add-on)
API BandwidthStandard bandwidthEnhanced (higher bandwidth) access to Office 365 Management Activity API ⭐
Intelligent InsightsNot availableIntelligent insights on user mail item activities and searches ⭐ — supports forensic investigations
Access to audit eventsStandard set of eventsAdditional high-value events for deeper investigation
📌
Exam tip — Audit Premium distinguishing features "Higher bandwidth access to audit logs compared to Audit Standard" → Enhanced Office 365 Management Activity API bandwidth (Audit Premium)
"Unique capability that supports forensic investigations" → Intelligent insights on user mail interactions and searches (Audit Premium)
"Facilitate compliance investigations compared to Standard" → Intelligent insights into mail item activities and user searches

💬 Communication Compliance

Communication Compliance in Microsoft Purview helps organizations detect, capture, and act on inappropriate messages across email, Microsoft Teams, and Yammer. It helps enforce corporate communication standards and regulatory requirements.

Predefined PoliciesBuilt-in templates (harassment, adult content, regulatory)
Custom PoliciesOrg-defined rules targeting specific keywords or topics
Designated ReviewersCompliance team members who examine flagged messages and take action
Corrective ActionsEscalate, notify user, remediate, or escalate to HR/Legal
📌
Exam tip — Communication Compliance scenarios "Messages reviewed and flagged for potential compliance violations related to harassment" → Communication Compliance with predefined and custom policies
"Detects inappropriate messages and allows designated reviewers to take corrective actions" → Communication Compliance
"Helps organizations adhere to corporate message standards" → Communication Compliance (it detects, humans review and act — it does NOT automatically resolve issues)
Microsoft Priva — Privacy Management

Microsoft Priva helps organizations safeguard personal data and build a privacy-resilient workplace. It discovers where personal data lives across M365, identifies privacy risks, and enables subject rights request management.

Privacy Risk MgmtIdentifies risks like data overexposure or data transfers
Subject Rights RequestsAutomates GDPR/CCPA data access requests
Service Trust PortalMicrosoft's transparency hub — audit reports, compliance docs
⚠️
Purview vs Priva — easy to mix upPurview = protects organizational data (sensitivity labels, DLP, compliance, eDiscovery — think "protecting company secrets"). Priva = protects individuals' personal data (GDPR/CCPA rights requests, privacy risk — think "protecting people's private info"). Both live in the Purview portal, but they solve different problems.
