A
Add-On License
An optional license that adds specific capabilities on top of a user's base M365 plan. Add-ons are assigned per user and require a qualifying base plan — examples include Microsoft 365 Copilot, Teams Phone, and Microsoft Defender P2.
D4→ Domain 4
Agility
A cloud benefit describing the ability to provision new resources — virtual machines, storage, services — in minutes rather than weeks. Enables organisations to respond rapidly to changing business needs without procurement delays.
D1→ Domain 1
Annual Commitment
A Microsoft 365 billing option where the customer commits to a 12-month subscription. Offers a lower per-user monthly cost than month-to-month billing. Can be paid monthly in instalments or as a single annual upfront payment.
D4→ Domain 4
Auditing Purview feature
A Microsoft Purview feature that maintains a unified audit log recording user and admin activities across all M365 services — file access, permission changes, sign-in events, and more. Used for security investigations and compliance reporting.
D3→ Domain 3
Authentication
The process of verifying who you are — confirming identity before granting access. In M365, handled by Microsoft Entra ID through password verification, MFA prompts, and biometrics. Distinct from authorisation (what you can access once authenticated).
D3→ Domain 3
Authorization
The process of determining what an authenticated user can access — which files, apps, and resources they have permission to use. In M365, controlled by Entra ID roles, SharePoint permissions, and Conditional Access policies.
D3→ Domain 3
⭐ Azure Virtual Desktop AVD
A cloud-based Windows desktop virtualisation service running on Azure infrastructure. Unlike Windows 365, AVD supports multi-session Windows (multiple users per VM), uses pay-as-you-go pricing, and requires IT to configure and manage the Azure infrastructure. Best for large enterprises optimising cost at scale.
D2→ Domain 2
B
Base License
The foundational M365 subscription plan a user must have before any add-ons can be assigned — e.g. M365 Business Premium or M365 E3. Every licensed user needs a base plan; add-ons cannot be purchased standalone.
D4→ Domain 4
Microsoft Bookings
An M365 app that provides a public-facing booking page where external clients can self-schedule appointments with staff. Automatically syncs with the team's Outlook calendars and sends confirmations. Designed for customer-facing scheduling (clinics, consultations, etc.).
D2→ Domain 2
C
⭐ CapEx Capital Expenditure
A large, upfront investment in physical infrastructure — servers, networking equipment, on-premises datacentres. Traditional IT used CapEx heavily. Cloud computing shifts this to OpEx (ongoing operational costs), eliminating large one-time hardware purchases.
D1→ Domain 1
⭐ Cloud Solution Provider CSP
A Microsoft purchasing channel where organisations buy M365 licenses through a certified Microsoft partner. The partner handles billing, invoicing, and first-line support. Best for SMBs wanting a managed service relationship. The CSP escalates to Microsoft only if they cannot resolve an issue.
D4→ Domain 4
Co-management
A Microsoft Intune capability that allows devices to be managed by both Intune (cloud) and on-premises Configuration Manager simultaneously. Enables organisations to migrate from traditional on-prem device management to cloud gradually, without a hard cutover.
D2→ Domain 2
⭐ Conditional Access
A Microsoft Entra ID policy engine that evaluates contextual signals — user identity, device compliance, location, app type, sign-in risk — and enforces access conditions dynamically. Example: "If signing in from an unknown country, require MFA." More flexible than blanket MFA; applies tighter controls only when risk signals are present.
D3→ Domain 3
⭐ Compliance Manager
A Microsoft Purview dashboard that scores an organisation's compliance posture against specific regulatory frameworks — GDPR, ISO 27001, NIST, HIPAA, and more. Provides a compliance score, maps M365 settings to regulatory controls, and recommends improvement actions. Distinct from Secure Score: Secure Score = security posture; Compliance Manager = regulatory compliance posture.
D3→ Domain 3
Consumption-Based Billing
A cloud pricing model where customers pay only for what they use — compute hours, storage consumed, API calls made. Contrasts with buying fixed hardware upfront. M365 SaaS plans use per-user subscription pricing rather than pure consumption billing.
D1→ Domain 1
Copilot Chat
A web-grounded, enterprise-safe AI chat experience available to all M365 users (including those without a Copilot add-on license). Unlike full M365 Copilot, Copilot Chat uses public web data rather than the organisation's internal M365 data. Accessible via Microsoft Teams and the browser.
D2→ Domain 2
D
Data Residency
A Microsoft Purview feature that ensures an organisation's M365 customer data is stored in specific geographic regions (e.g. EU-only datacentres). Helps meet regulatory requirements such as GDPR that mandate data stays within certain borders.

Related: Data Sovereignty refers to the legal principle that data is subject to the laws of the country where it's stored. Data Residency is Microsoft's technical implementation of data sovereignty commitments. The MS-900 exam tests "data residency" — data sovereignty is the underlying concept that explains why it matters.
D3→ Domain 3
Defender for Cloud Apps
A Microsoft Defender XDR product that discovers Shadow IT (unsanctioned SaaS apps employees use without IT approval), enforces session controls on cloud apps, and monitors user behaviour across cloud services. Protects the cloud app attack surface.
D3→ Domain 3
⭐ Defender for Endpoint
A Microsoft Defender XDR product that protects devices (Windows, macOS, Linux, iOS, Android) through Endpoint Detection and Response (EDR), threat analytics, and vulnerability management. Detects malware, ransomware, and suspicious process behaviour on the device itself.
D3→ Domain 3
Defender for Identity
A Microsoft Defender XDR product that monitors on-premises Active Directory for signs of compromised identities and lateral movement attacks. Detects pass-the-hash, pass-the-ticket, reconnaissance, and other AD-specific attack patterns. Note: protects on-prem AD, not cloud-only Entra ID.
D3→ Domain 3
⭐ Defender for Office 365
A Microsoft Defender XDR product that protects email and M365 collaboration tools (Outlook, Teams, SharePoint, OneDrive) from phishing, malware, and malicious links. Key features include Safe Links (rewrites and checks URLs at click time) and Safe Attachments (detonates attachments in a sandbox before delivery).
D3→ Domain 3
⭐ Defender XDR Extended Detection & Response
Microsoft's unified security platform that correlates threat signals from all Defender products — endpoint, email, identity, and cloud apps — into a single incident view in the Microsoft Defender Portal. Enables analysts to see the full attack chain rather than isolated product-by-product alerts.
D3→ Domain 3
Microsoft Defender Portal
The unified admin interface for all Microsoft Defender security products, accessed at security.microsoft.com. Admins investigate incidents, review alerts, manage Microsoft Secure Score, run threat hunting, and configure all four Defender products from a single pane. Separate from the Purview compliance portal (compliance.microsoft.com) and the M365 Admin Center (admin.microsoft.com).
D3→ Domain 3
⭐ Deployment Rings
A Windows-as-a-Service (WaaS) mechanism for controlling when Windows updates reach different groups of devices. Three rings: Insider/Preview (early testers) → Pilot (representative production users) → Broad (entire organisation). Allows IT to catch issues before updates reach all devices.
D2→ Domain 2
Distribution Group
An Exchange Online object that acts as an email alias routing to multiple recipients. Sending to "sales@contoso.com" delivers to everyone in the Sales distribution group. Unlike Microsoft 365 Groups, distribution groups are email-only — they don't include Teams, SharePoint, or shared mailboxes.
D2→ Domain 2
⭐ DLP Data Loss Prevention
A Microsoft Purview feature that detects and prevents the sharing of sensitive data — such as credit card numbers, national IDs, or health records — across M365 services. DLP policies can block emails, show warning prompts, or log incidents when sensitive content is detected.
D3→ Domain 3
E
eDiscovery
A Microsoft Purview feature enabling organisations to search, preserve, and export content across M365 (emails, Teams chats, SharePoint documents) for legal proceedings or internal investigations. Supports legal hold — preventing content from being deleted while under investigation.
D3→ Domain 3
Elasticity
A cloud benefit describing the ability to automatically scale resources both up and down in response to demand. Distinct from scalability (which is just scaling up/out). Elasticity means paying for more at peak times and automatically releasing unused resources when demand drops.
D1→ Domain 1
Endpoint Analytics
A Microsoft Intune feature providing reporting on device health, startup performance, and application reliability. Helps IT identify devices with slow boot times or frequent crashes before users report issues. Part of the Microsoft Intune endpoint management suite.
D2→ Domain 2
⭐ Enterprise Agreement EA
A Microsoft purchasing channel designed for organisations with 500+ users. Provides volume licensing discounts in exchange for a 3-year commitment with annual payments directly to Microsoft. Offers the lowest per-unit cost but requires long-term commitment and larger upfront planning.
D4→ Domain 4
Entra Connect
The Microsoft synchronisation tool that bridges on-premises Active Directory and Microsoft Entra ID for hybrid identity. Syncs user accounts, groups, and passwords so employees use one set of credentials on-prem and in the cloud. Formerly called Azure AD Connect.
D3→ Domain 3
Exchange Online
Microsoft's cloud-hosted email and calendaring service. The backend that powers Outlook mailboxes, shared calendars, distribution groups, and resource booking in M365. Outlook (desktop/web/mobile) is the client used to access Exchange Online. Included in all M365 plans.
D2→ Domain 2
F
Financially-Backed SLA
Microsoft's formal, contractual uptime commitment. If Microsoft fails to meet the published SLA (e.g. 99.9% uptime), customers can claim service credits as financial compensation. "Financially-backed" distinguishes these from informal promises — the commitment is written into the service agreement.
D4→ Domain 4
Microsoft Forms
An M365 app for creating surveys, quizzes, polls, and data collection forms. Results are collected in real-time and can be exported to Excel. Commonly used for feedback, assessments, event registrations, and quick polls. Distinct from Microsoft Lists (structured data) and Bookings (appointment scheduling).
D2→ Domain 2
G
⭐ Microsoft 365 Groups
The underlying identity object that powers team collaboration in M365. When you create a Microsoft Team, a shared mailbox, or a SharePoint team site, an M365 Group is automatically created behind the scenes — it defines the membership and ties together the shared inbox (Exchange), file storage (SharePoint), and Teams workspace. Groups are managed in Entra ID. Distinct from Distribution Groups (email-only, no Teams/SharePoint backend).
D2→ Domain 2
H
High Availability
A cloud benefit describing a service's ability to remain operational and accessible even during failures. Microsoft guarantees high availability via SLAs (e.g. 99.9% uptime for M365) through geo-redundant datacentres, automatic failover, and load balancing.
D1→ Domain 1
⭐ Hybrid Cloud
A cloud deployment model that combines on-premises infrastructure with a public cloud (e.g. Azure), connected so they can share data and workloads. Common in enterprise M365 deployments where organisations sync on-prem Active Directory to Entra ID while keeping some systems on-premises.
D1→ Domain 1
Hybrid Identity
An identity model where on-premises Active Directory accounts are synchronised to Microsoft Entra ID via Entra Connect. Employees use a single set of credentials that works both on the corporate network and in cloud services. The most common identity model for established enterprises migrating to M365.
D3→ Domain 3
I
⭐ IaaS Infrastructure as a Service
A cloud service model where Microsoft provides virtualised compute, networking, and storage infrastructure, while the customer manages the OS, middleware, applications, and data. Example: Azure Virtual Machines. Gives the most control, requires the most management. M365 is not IaaS — it's SaaS.
D1→ Domain 1
Insider Risk Management
A Microsoft Purview feature that uses machine learning to detect risky user behaviours within the organisation — such as unusual file downloads, data exfiltration before departure, or policy violations. Analyses M365 signals to surface potential insider threats while respecting privacy controls.
D3→ Domain 3
⭐ Microsoft Intune
Microsoft's cloud-based Mobile Device Management (MDM) and Mobile Application Management (MAM) solution. Allows IT to enrol and manage devices (Windows, iOS, Android, macOS), enforce compliance policies, deploy apps, and wipe corporate data remotely. Included in Microsoft 365 Business Premium, E3, and E5.
D2→ Domain 2
L
License Assignment
The administrative act of granting a specific M365 license to a user, performed in the Microsoft 365 Admin Center. Until a license is assigned, the user cannot access the corresponding M365 services. Licenses can be assigned individually or via group-based licensing in Entra ID.
D4→ Domain 4
License Reclamation
The process of removing a license from a user who no longer needs it — typically when an employee leaves — and reassigning it to a new user. M365 licenses are not consumed permanently; reclaiming unused licenses reduces ongoing subscription costs.
D4→ Domain 4
Microsoft Lists
An M365 app for creating structured, configurable lists to track data — assets, issues, contacts, inventory. Built on SharePoint, Lists supports custom columns, views, conditional formatting, and Power Automate integrations. Think smart spreadsheet for team tracking, not personal tasks (To Do) or visual boards (Planner).
D2→ Domain 2
Microsoft Loop
An M365 app providing portable, real-time collaborative workspaces — Loop components (tables, lists, notes) can be embedded in Teams, Outlook, and Word and stay in sync everywhere. Designed for fluid, cross-app collaboration where content follows the team rather than living in one place.
D2→ Domain 2
M
⭐ MAM Mobile Application Management
A Microsoft Intune capability that manages and protects corporate apps on a device without enrolling the device itself. Ideal for BYOD — IT can enforce policies on the Outlook or Teams app (requiring PIN, blocking copy-paste) without touching the employee's personal phone data.
D2→ Domain 2
⭐ MDM Mobile Device Management
A Microsoft Intune capability that provides full management of enrolled devices — enforcing encryption, compliance policies, app deployment, and remote wipe. Requires the device to be enrolled (registered) with Intune. Used for corporate-owned devices. Compare with MAM, which doesn't require device enrolment.
D2→ Domain 2
Message Center
A section of the Microsoft 365 Admin Center that communicates planned service changes, new features, and maintenance notifications from Microsoft. Distinct from Service Health (which reports live incidents). Admins use the Message Center to stay ahead of upcoming changes that may require preparation.
D2→ Domain 2
⭐ MFA Multi-Factor Authentication
A security mechanism requiring users to verify identity with two or more factors: something you know (password), something you have (phone/authenticator app), or something you are (biometrics). Prevents account takeover even when passwords are stolen — the attacker can't satisfy the second factor.
D3→ Domain 3
Microsoft 365 M365
Microsoft's full productivity, security, and device management bundle. Includes everything in Office 365 (apps + cloud services) plus Microsoft Intune (device management), advanced security features, and (in E3/E5 plans) Windows 11 Enterprise licensing. Think: Office 365 + Intune + Security.
D1→ Domain 1
⭐ Microsoft 365 Copilot
An AI assistant embedded across M365 apps — Word, Excel, Teams, Outlook, PowerPoint. Uses Microsoft Graph to access the user's own organisational data (emails, documents, meetings) to generate contextually relevant drafts, summaries, and insights. Requires an add-on license on top of a qualifying base plan. Respects existing M365 permissions — it won't surface content the user can't already access.
D2→ Domain 2
⭐ Microsoft Entra ID
Microsoft's cloud-based identity and access management service — the identity backbone of every M365 tenant. Formerly known as Azure Active Directory (Azure AD). Handles authentication, single sign-on (SSO), MFA, Conditional Access, and user/group management for all M365 and Azure services.
D3→ Domain 3
Microsoft Graph
Microsoft's unified API and data connection layer that links all M365 services — emails, calendar, Teams messages, files, contacts, and more — into a single programmable interface. Powers M365 Copilot by providing contextual organisational data. Also enables developers to build apps that integrate with M365.
D2→ Domain 2
Microsoft Planner
An M365 app providing Kanban-style team task boards with "To Do / In Progress / Done" columns. Designed for team-level task tracking. Can be added as a Teams tab. Distinct from Microsoft To Do (personal tasks) and Microsoft Project (complex enterprise project management).
D2→ Domain 2
Microsoft Priva
A Microsoft solution for personal data privacy management. Discovers where personal data lives across M365, identifies privacy risks (overexposure, unexpected transfers), and automates handling of GDPR/CCPA subject rights requests. Contrast with Purview: Priva protects individuals' data; Purview protects organisational data.
D3→ Domain 3
⭐ Microsoft Purview
The umbrella brand for Microsoft's information protection and compliance suite. Covers Sensitivity Labels, Data Loss Prevention (DLP), Insider Risk Management, eDiscovery, Auditing, and Data Residency. Accessible via the Purview compliance portal. Protects organisational data and ensures regulatory compliance.
D3→ Domain 3
⭐ Microsoft Teams
Microsoft's primary collaboration and communication hub in M365, integrating chat, video meetings, voice calling (Teams Phone), file sharing, and extensibility through apps. Files are stored in SharePoint; meeting recordings go to Stream; Exchange powers meeting invites. The central interface for most M365 workloads.
D2→ Domain 2
Microsoft To Do
An M365 app for managing personal task lists and individual to-do items. Integrates with Outlook (flagged emails become tasks) and Planner. Designed for individual use — contrast with Planner (team Kanban boards) and Lists (structured team data tracking).
D2→ Domain 2
Microsoft Whiteboard
A digital infinite canvas for visual brainstorming and collaboration, available as a Teams extension during meetings. Participants can draw, add sticky notes, and annotate in real time. Designed to replicate physical whiteboard sessions in remote or hybrid meetings.
D2→ Domain 2
O
Office 365
Microsoft's cloud productivity suite — a subset of Microsoft 365. Includes core apps (Word, Excel, PowerPoint, Outlook, Teams) and cloud services (Exchange Online, SharePoint, OneDrive). Does not include Intune (device management) or the advanced security features added in Microsoft 365. Being phased out as a standalone brand.
D1→ Domain 1
⭐ OneDrive
Microsoft's personal cloud file storage service — each user gets their own OneDrive storage space for individual files that can sync to the desktop and be selectively shared. Contrast with SharePoint (team/organisational file storage). Teams channel files are stored in SharePoint, not OneDrive.
D2→ Domain 2
⭐ OpEx Operational Expenditure
Ongoing, predictable operational costs — like monthly subscription fees. Cloud computing shifts IT spend from CapEx (large upfront hardware) to OpEx (recurring subscription), improving cash flow and allowing costs to scale with actual usage.
D1→ Domain 1
OWA Outlook Web App
The browser-based version of Outlook for accessing Exchange Online email and calendar without installing the desktop app. Available at outlook.office.com. Functionally equivalent to the desktop app for most tasks. Also referred to as Outlook on the web.
D2→ Domain 2
P
⭐ PaaS Platform as a Service
A cloud service model where Microsoft manages the infrastructure and runtime platform, and the customer manages only their application code and data. Example: Azure App Service. Removes OS and middleware management burden. M365 is not PaaS — it's SaaS (Microsoft manages the entire application).
D1→ Domain 1
Power Apps
A Microsoft platform for building custom no-code/low-code business applications that can be embedded as Teams tabs. Allows non-developers to create tailored tools — leave request forms, inventory trackers, approval workflows — without writing traditional code. Part of the Microsoft Power Platform.
D2→ Domain 2
Power Automate
A Microsoft platform for building automated workflows triggered by events across M365 and third-party services. Examples: automatically post to a Teams channel when a new Form response arrives; send an approval email when a file is uploaded to SharePoint. Can be integrated directly into Teams. Part of the Microsoft Power Platform.
D2→ Domain 2
Privacy Risk Management
A Microsoft Priva feature that identifies and surfaces privacy risks in M365 — such as personal data being overexposed (shared too broadly), transferred to unexpected locations, or held beyond retention limits. Generates alerts and policy recommendations to help organisations proactively reduce privacy exposure.
D3→ Domain 3
Private Cloud
A cloud deployment model where the infrastructure is owned and used exclusively by a single organisation. Can be hosted on-premises in the organisation's own datacentre or with a third party. Provides maximum control and customisation but at higher cost — no sharing of infrastructure with other tenants.
D1→ Domain 1
⭐ Public Cloud
A cloud deployment model where infrastructure is owned and operated by a cloud provider (e.g. Microsoft Azure) and shared across multiple customers (multi-tenant). Highly scalable and cost-effective. Microsoft 365 is delivered from the public cloud — organisations don't own the underlying infrastructure.
D1→ Domain 1
R
⭐ RBAC Role-Based Access Control
A security model used in Microsoft Entra ID to grant admins only the permissions their role requires — no more. Key M365 admin roles: Global Admin (full control of all M365 — use sparingly), User Admin (create/manage users and assign licenses), Billing Admin (manage subscriptions and payments), Security Admin (configure Defender and security settings), SharePoint Admin (manage SharePoint sites). Exam principle: always assign the least privileged role that allows the task to be done.
D3→ Domain 3
Reliability
A cloud benefit describing a service's resilience to failures through fault tolerance and geographic redundancy. Microsoft runs M365 across multiple datacentres in multiple regions — if one fails, traffic automatically routes to another. Enables the high availability SLA guarantees.
D1→ Domain 1
S
⭐ SaaS Software as a Service
A cloud service model where Microsoft manages everything — infrastructure, OS, runtime, and the application itself. The customer manages only their data and users. Microsoft 365 is SaaS. No servers to maintain, no OS patches to apply — Microsoft handles it all. Maximum abstraction, minimum customer management burden.
D1→ Domain 1
Scalability
A cloud benefit describing the ability to increase resources to handle growing demand — scaling up (more powerful hardware) or scaling out (more instances). Unlike elasticity (which also scales down), scalability focuses on the upward growth capacity. M365 scales globally with no capacity limits for the customer to manage.
D1→ Domain 1
⭐ Microsoft Secure Score
A numeric measurement (0–100+) of an organisation's M365 security posture. Points are earned for each recommended security action completed. Provides a prioritised list of actions with estimated score improvements, and enables benchmarking against similar organisations. Accessed via the Microsoft Defender Portal.
D3→ Domain 3
⭐ Sensitivity Labels
A Microsoft Purview feature that allows organisations to classify and protect documents and emails by applying labels (e.g. "Public", "Confidential", "Highly Confidential"). Labels can automatically apply encryption, add watermarks, restrict forwarding, and control sharing permissions. Labels travel with the document wherever it goes.
D3→ Domain 3
⭐ Service Credit
A billing credit applied to future M365 invoices when Microsoft fails to meet its SLA (e.g. if uptime drops below 99.9%). Critically, service credits are not cash refunds — they offset future charges. The credit percentage scales with the severity of the outage.
D4→ Domain 4
Service Trust Portal
Microsoft's public transparency and compliance documentation hub at servicetrust.microsoft.com. Contains third-party audit reports, compliance certifications (ISO 27001, SOC 2, FedRAMP, GDPR), data protection documentation, and whitepapers. Used by organisations to verify Microsoft's compliance posture.
D3→ Domain 3
Shadow IT
Cloud apps and services used by employees without IT approval — e.g. uploading files to a personal Dropbox, using a free project tool, or connecting an unapproved app to company data. Shadow IT creates security and compliance risks. Microsoft Defender for Cloud Apps discovers and helps govern shadow IT.
D3→ Domain 3
⭐ Shared Responsibility Model
The framework defining how security responsibilities are divided between Microsoft and the customer depending on the service model. Microsoft always manages physical infrastructure. Customers always manage their data and identities. Who manages the OS and applications depends on whether it's IaaS, PaaS, or SaaS. Key exam rule: data and identities are always the customer's responsibility.
D1→ Domain 1
⭐ SharePoint
Microsoft's organisational content management and intranet platform. Provides document libraries, team sites, intranet pages, and workflow automation. Every Microsoft Teams team has a corresponding SharePoint site for file storage. Contrast with OneDrive (personal files) and Stream (video).
D2→ Domain 2
⭐ SLA Service Level Agreement
Microsoft's contractual uptime guarantee for each M365 service — typically 99.9%, equating to ~44 minutes maximum downtime per month. If Microsoft breaches the SLA, organisations can claim service credits. The SLA is "financially-backed" — it's a formal commitment, not a best-effort promise.
D4→ Domain 4
SSO Single Sign-On
An authentication capability where users sign in once and gain access to all connected applications without re-entering credentials. In M365, Entra ID provides SSO across all M365 apps, Azure services, and thousands of third-party SaaS apps. Improves user experience and reduces password fatigue.
D3→ Domain 3
⭐ SSPR Self-Service Password Reset
A Microsoft Entra ID feature allowing users to reset their own passwords without contacting the helpdesk, using pre-registered verification methods (email, phone, authenticator app, security questions). Reduces IT support tickets and prevents productivity loss from lockouts. Distinct from MFA (which secures sign-in) and Conditional Access (which governs access policies).
D3→ Domain 3
Microsoft Stream
Microsoft's cloud video hosting platform in M365. Teams meeting recordings are automatically saved to Stream (stored in OneDrive or SharePoint). Supports video search via transcripts, sharing within the organisation, and channels for organising videos. Video-only — not a document or file storage service.
D2→ Domain 2
Subject Rights Requests
A Microsoft Priva feature that automates the handling of GDPR/CCPA data subject requests — when individuals ask an organisation to provide, correct, or delete their personal data. Priva searches across M365 for relevant data, manages the review workflow, and tracks response deadlines to ensure regulatory compliance.
D3→ Domain 3
T
Teams Chat
Persistent 1:1 and group instant messaging within Microsoft Teams. Unlike channel conversations (which are project-topic-based), chats are direct conversations between individuals or small groups. Supports file sharing, reactions, GIFs, and Loop components. Chat history is searchable and persists indefinitely.
D2→ Domain 2
Teams Meetings
Video and audio conferencing within Microsoft Teams, supporting recording, live transcription, breakout rooms, polls, and screen sharing. Recordings are saved to Microsoft Stream. Available to internal and external (guest) participants via a browser or the Teams app. Also supports large-scale webinars and live events.
D2→ Domain 2
Teams Phone
A Microsoft Teams capability (and add-on license) that enables PSTN (public telephone network) calling directly from Teams — replacing traditional office PBX phone systems. Users can make and receive external calls using their Teams app. Requires a Teams Phone add-on license and a calling plan or direct routing setup.
D2→ Domain 2
Tenant
An organisation's dedicated, isolated instance of Microsoft Entra ID and M365 services. When a company signs up for M365, Microsoft provisions a tenant — a logically separated environment containing all the org's users, groups, data, and settings. One organisation = one tenant (typically). Tenants are globally identified by a unique domain (e.g. contoso.onmicrosoft.com).
D3→ Domain 3
V
Microsoft Viva
Microsoft's employee experience platform, integrated into Teams. A suite of apps designed to improve engagement, learning, wellbeing, and knowledge sharing. Includes Viva Insights, Connections, Learning, Goals, and Engage. Viva apps surface inside Teams so employees don't need to switch between separate platforms.
D2→ Domain 2
Viva Connections
A Viva app that brings the company intranet and internal communications directly into Microsoft Teams. Employees see company news, announcements, HR resources, and links to tools — without leaving Teams. Built on SharePoint as the content backend.
D2→ Domain 2
Viva Engage
A Viva app providing enterprise social networking and community features — formerly Yammer. Enables org-wide and interest-based community conversations, leadership Q&As, and knowledge sharing beyond formal team structures. Integrated into Teams as a communities platform.
D2→ Domain 2
Viva Goals
A Viva app for OKR (Objectives and Key Results) tracking and goal alignment across the organisation. Teams set objectives and track key results, with progress visible to managers and aligned to organisational goals. Integrates with Teams for weekly check-ins and progress updates.
D2→ Domain 2
Viva Insights
A Viva app providing personal and organisational productivity and wellbeing analytics — meeting hours, focus time, after-hours work, collaboration patterns. Individuals see their own data privately; managers see aggregated (anonymised) team data. Helps identify burnout risk and improve work habits.
D2→ Domain 2
Viva Learning
A Viva app that integrates learning content directly into Microsoft Teams — surfacing courses from LinkedIn Learning, Microsoft Learn, third-party LMSes, and internal content. Employees can discover, share, and complete training without leaving Teams. Managers can assign and track learning.
D2→ Domain 2
W
WaaS Windows-as-a-Service
Microsoft's continuous delivery model for Windows updates — replacing the old "major version every few years" approach. Feature updates are delivered on a regular cadence. Organisations manage rollout via deployment rings (Insider → Pilot → Broad) to control which devices receive updates when. Reduces large migration projects.
D2→ Domain 2
⭐ Windows 365
Microsoft's Cloud PC service — a dedicated, personal Windows desktop streamed to any device. SaaS-like: Microsoft manages all infrastructure, fixed per-user monthly pricing, no Azure configuration required. Best for SMBs and predictable workloads. Contrast with Azure Virtual Desktop (multi-session, pay-as-you-go, IT-managed infrastructure).
D2→ Domain 2
Windows Autopatch
A Microsoft Intune feature that automates Windows and Microsoft 365 Apps update management across a device fleet. IT enrols devices, and Autopatch handles update scheduling, deployment ring progression, and monitoring — removing the manual effort of managing patches at scale. Reduces update-related helpdesk burden.
D2→ Domain 2
⭐ Windows Autopilot
A Microsoft deployment technology enabling zero-touch device provisioning — new devices can be shipped directly to employees who configure them on first boot, with corporate policies automatically applied via Entra ID registration and Intune MDM enrolment. IT never needs to physically touch the device.
D2→ Domain 2
Z
⭐ Zero Trust
A security philosophy replacing the "castle and moat" model — instead of trusting all traffic inside the network, Zero Trust assumes breach and verifies every request explicitly. Three principles: Verify Explicitly (always authenticate using all signals), Least Privilege Access (minimum permissions needed), Assume Breach (design for containment, encrypt, segment). MFA and Conditional Access are practical Zero Trust implementations.
D3→ Domain 3
