Cloud Concepts

0 / 8
Q1
A hospital IT team wants to move their legacy patient records application to the cloud but cannot modify any application code due to regulatory constraints. The application must run exactly as it does today. Which migration strategy should they use?
D1
Rehost (lift and shift) moves an application to cloud VMs (IaaS) with zero code changes — exactly what is needed when modification is not possible. Refactor requires code changes. Repurchase replaces the app with SaaS (e.g. M365). Retire means shutting the app down. When the exam says "no code changes" → always think IaaS rehost.
Q2
Contoso Ltd currently budgets $500,000 per year for new server hardware. After migrating to Microsoft Azure, they will pay only for the computing resources they actually consume each month. Which cloud financial concept does this transition represent?
D1
Cloud shifts spending from CapEx (large upfront hardware investment) to OpEx (pay-as-you-go monthly consumption). CapEx = buy servers. OpEx = rent compute. Economies of scale is a real cloud benefit (Microsoft passes savings to customers) but describes the mechanism behind lower costs, not the spending model shift itself.
Q3
A start-up deploys a web application on Azure App Service. They write and deploy their code, but Microsoft manages the underlying OS, runtime patching, and server hardware. Which cloud service model is this?
D1
PaaS (Platform as a Service) — customers bring their own application code; the provider (Microsoft) handles OS, runtime, patching, and hardware. Azure App Service, Azure SQL Database, and Azure Functions are all PaaS examples. IaaS gives OS access to the customer. SaaS means the provider manages the entire application (e.g., Teams, Outlook.com).
Q4
An organisation runs critical workloads on Azure but must keep a subset of sensitive data on their own on-premises servers due to regulatory requirements. Some applications connect to both environments. Which cloud deployment model describes this?
D1
Hybrid cloud combines an organisation's on-premises (private) infrastructure with a public cloud (Azure), with connectivity between them. This is common for data residency or compliance requirements. Multi-cloud uses multiple public providers (Azure + AWS) but doesn't involve on-premises servers. This distinction is frequently tested — "some on-prem, some cloud" = hybrid.
Q5
Contoso's e-commerce website experiences a traffic spike every Black Friday that is 10× normal load. They want to automatically add servers during the spike and remove them afterward, paying only for what they use. Which cloud benefit does this describe?
D1
Elasticity is the ability to automatically scale compute resources up or down in response to demand — and pay only for what you use. High availability focuses on uptime/redundancy (keeping the site running), not scaling. Geo-redundancy is about data replication across regions. Disaster recovery is about recovering from failures. Spike + auto-scale + pay-as-you-go = elasticity.
Q6
A company's on-premises CRM software is approaching end-of-life. Instead of upgrading the on-premises version, they decide to switch to Microsoft Dynamics 365 (a cloud-based CRM). Which migration strategy does this represent?
D1
Repurchase (also called "drop and shop") means replacing an existing on-premises application with a SaaS equivalent. Switching from an on-prem CRM to Dynamics 365 (or from on-prem Exchange to Exchange Online / Microsoft 365) is the classic repurchase example. The exam tests this frequently — "replace with SaaS" = repurchase. Rehost = same app, cloud VM. Refactor = same app, rewritten for cloud.
Q7
Which of the following is a key advantage that Microsoft can offer cloud customers due to economies of scale?
D1
Economies of scale means that because Microsoft operates datacenters at massive global scale, they can purchase hardware, electricity, and networking at far lower per-unit costs than any individual organisation — and pass those savings on to customers as lower pricing. No cloud provider offers 100% uptime (SLAs are typically 99.9%+). Customers don't own physical hardware. Storage is priced by consumption.
Q8
Under the shared responsibility model for Microsoft 365 (SaaS), which of the following is ALWAYS the customer's responsibility regardless of service model?
D1
In the shared responsibility model, user identities and access management are always the customer's responsibility — regardless of whether the model is IaaS, PaaS, or SaaS. Microsoft manages OS patching and physical hardware for SaaS (M365). Middleware is managed by Microsoft in SaaS and PaaS. But the customer always controls who has accounts and what permissions they hold — this cannot be outsourced to Microsoft.

Q9

Microsoft Loop introduces portable "Loop components" — live, co-editable blocks (tables, task lists, paragraphs) that stay in sync across Teams chats, Outlook emails, and the Loop app. Whiteboard is for visual brainstorming. D2
Power BI is Microsoft's self-service business intelligence platform. It connects to hundreds of data sources, transforms data, and publishes interactive visual reports and dashboards accessible via browser or mobile app — no Excel or local software required for viewers. Excel is for spreadsheet analysis. Visio is for diagrams. Project is for scheduling. Power BI is the answer whenever the scenario involves "dashboard," "BI," or "data visualisation."
Q11

D2
Microsoft Forms allows creation of surveys and forms that can be shared publicly (no Microsoft account required). Responses are collected in real time and can sync to Excel or SharePoint. Forms integrates with Power Automate for workflow automation. Lists is an internal structured-data tool. Planner is for tasks. Bookings is for appointment scheduling — not arbitrary data collection from vendors.
Q12
Northwind Traders' HR team wants to let employees book 30-minute onboarding slots with HR staff, showing real availability based on the HR staff's Outlook calendar. Which Microsoft 365 app is purpose-built for this?
D2
Microsoft Bookings is a scheduling app that publishes a booking page showing real staff availability (synced with Outlook calendars) and lets customers/employees self-book appointments. It sends automatic reminders and can integrate with Teams for virtual meetings. Forms collects data but doesn't manage calendar availability. Planner is for tasks. To Do is a personal task manager.
Q13

Microsoft 365 Frontline Worker plans (F1 and F3) are specifically designed for deskless/shift workers who don't have dedicated PCs. F1 is web-and-mobile only (no desktop Office apps); F3 adds desktop apps and more. Q14
An organisation wants to use Microsoft Copilot to draft emails in Outlook, summarise Teams meetings, and generate first drafts in Word — all powered by their own Microsoft 365 data with enterprise privacy protections. What is required in addition to their existing Microsoft 365 E3 licences?
D2
Microsoft 365 Copilot is an add-on licence that must be purchased per user on top of a qualifying Microsoft 365 base plan (E3, E5, Business Premium, etc.). It is not included in any base plan — not even E5. It requires eligible base licensing (M365 E3/E5 or similar). No separate Azure subscription is needed; the Copilot service is fully integrated into Microsoft 365's infrastructure.
Q15

Teams (teams and channels) provide persistent workspaces where department members see ongoing channel conversations, shared files (backed by SharePoint), a OneNote wiki, Planner tabs, and other integrated apps — all in one structure. Chat is ad-hoc messaging without the persistent team structure. Calls handles telephony. Live Events is for one-to-many broadcasts. For structured departmental workspaces → Teams & Channels.
Q16
Woodgrove Bank needs to automate a multi-step approval process: when a sales rep submits an expense report in SharePoint, it should notify their manager in Teams, wait for approval, and if approved, update a record in their CRM — all without writing custom code. Which Microsoft 365 component enables this?
D2
Power Automate (formerly Flow) is the no-code/low-code workflow automation tool in the Power Platform. It connects triggers (SharePoint form submitted) to actions (send Teams notification, wait for approval, update CRM) using hundreds of pre-built connectors. Power Apps builds UIs. Power BI builds dashboards. Power Virtual Agents builds chatbots. Multi-step automated workflow without code = Power Automate.
Q17
An organisation's IT department wants to deploy Windows 11 updates to pilot users first, then to a broader group two weeks later, and finally to all remaining users — ensuring issues are caught before organisation-wide rollout. Which Windows as a Service concept describes this staged approach?
D2
Deployment Rings are a Windows as a Service (WaaS) strategy that stages update rollouts across groups: a small pilot ring first, then a broader early adopter ring, then the full organisation. This lets IT catch compatibility issues before they affect everyone. Windows Insider is Microsoft's pre-release preview program. Quality Updates are monthly security patches. Feature Updates are major Windows feature releases — deployment rings control the timing of all of these.
Q18
A non-profit organisation wants to give volunteers access to OneDrive, SharePoint, and Teams but does not need full desktop Office applications. Which Microsoft 365 plan provides core collaboration tools at the lowest cost?
D2
Microsoft 365 Business Basic is the entry-level plan providing Teams, SharePoint, OneDrive, Exchange (email), and web/mobile versions of Office apps — but no desktop Office installs. It's the most cost-effective option when desktop Office is not required. Business Standard adds desktop Office. E3 adds advanced enterprise security and compliance. Apps for Business provides desktop Office but no Teams or Exchange. "Web + Teams + OneDrive, lowest cost" = Business Basic.
Q19
Contoso wants to align all teams around measurable company objectives. They want employees to log their individual key results and see how their work connects to company-wide goals — using a Microsoft Viva module. Which module is designed for this OKR (Objectives and Key Results) use case?
D2
Viva Goals is the OKR (Objectives and Key Results) module in Microsoft Viva. It lets organisations define company objectives, cascade them to teams and individuals, and track progress — all integrated into Teams. Viva Insights surfaces wellbeing and productivity data. Viva Engage is the social networking layer (built on Yammer). Viva Learning surfaces training courses. OKR / goal alignment = Viva Goals.
Q20

SharePoint is Microsoft's platform for building intranets, publishing content, and managing documents. Modern SharePoint Communication Sites provide a polished intranet experience with news posts, page sections, document libraries, and search — all without coding. Teams is for real-time collaboration. Q21
An HR manager wants to take handwritten interview notes from a Teams meeting recording and have them automatically transcribed, summarised, and drafted into a candidate feedback document. Which Microsoft 365 Copilot capability in which app would best automate this?

Copilot in Teams can access meeting transcripts and recordings to produce summaries, identify key points, highlight decisions, and generate action items directly in the chat — without the user manually reviewing the full recording. Copilot in Word would help draft documents, but the transcription and meeting summary step happens in Teams. For meeting intelligence and follow-up automation → Copilot in Teams.
Q22
A company uses Microsoft Endpoint Manager (Intune) to manage all their corporate devices. They need to ensure that all Windows laptops have a specific security baseline applied before users can access company data. Which Intune capability handles this?
D2
Device Compliance Policies in Microsoft Intune define the minimum security requirements a device must meet (OS version, encryption enabled, PIN set, etc.). Devices that don't meet requirements are marked non-compliant and can be blocked from accessing corporate data (via Conditional Access). Autopilot handles device setup/provisioning. Defender for Endpoint is for threat protection. Azure Virtual Desktop delivers virtual machines to users.
Q23
An organisation wants to display company news, personalised learning suggestions, and employee wellbeing check-ins directly within Teams as a single employee experience platform. Which Microsoft product family consolidates these employee experience modules?

Microsoft Viva is the employee experience platform built on Microsoft 365 and Teams. Its modules include: Viva Connections (company news/intranet in Teams), Viva Learning (training), Viva Insights (wellbeing analytics), Viva Goals (OKRs), and Viva Engage (social). Power Platform is for low-code automation and app building. Dynamics 365 is for business applications (CRM/ERP). Syntex is for AI document processing.
Q24
Microsoft 365 E5 includes all the capabilities in E3 plus several advanced add-ons. Which of the following is a capability available in E5 but NOT in E3?

Microsoft 365 E5 adds advanced security and compliance on top of E3: Microsoft Defender for Office 365 Plan 2 (advanced threat protection), Microsoft Defender for Identity, Purview eDiscovery Premium, Insider Risk Management, Communication Compliance, and Power BI Pro. Teams, Exchange, and desktop Office apps are included in both E3 and E5. The E3→E5 jump is about security depth and compliance intelligence.
Q25

Q26

Microsoft Teams Phone (formerly called Phone System) is the cloud PBX capability in Microsoft 365 that enables PSTN calling directly from Teams — replacing traditional on-premises PBX systems. Q27
Contoso's security team wants to prevent a user from logging into Microsoft 365 from an unknown country unless they complete multi-factor authentication. Which Microsoft Entra ID feature should they configure?
D3
Conditional Access in Microsoft Entra ID (formerly Azure AD) evaluates signals (user, location, device, app, risk level) and enforces access controls — such as requiring MFA when signing in from an unfamiliar location or blocking access from certain countries entirely. PIM manages time-limited privileged roles. B2C is for consumer/external identity. SSPR lets users reset passwords. Location-based MFA enforcement = Conditional Access.
Q28
Fabrikam needs to allow their business partners at AdventureWorks (who use their own Azure AD tenant) to access a shared SharePoint site and Teams channel without creating guest accounts managed by Fabrikam. Which Microsoft Entra ID capability enables this cross-tenant collaboration?
D3
Azure AD B2B (Entra External ID for business partners) allows external users in another organisation's Azure AD tenant to access your resources using their own credentials — no need to manage separate guest passwords. The partner signs in with their own identity; Fabrikam grants them access. B2C is for consumer-facing apps (millions of anonymous customers). PIM is for admin role elevation. B2B = business-to-business partner collaboration across tenants.
Q29
A data protection officer needs to find all emails containing credit card numbers sent externally in the past 90 days, place a legal hold on them, and export them for a regulatory audit — without disturbing the users' normal mailbox access. Which Microsoft Purview feature handles this?
D3
eDiscovery (in Microsoft Purview) enables legal and compliance teams to search across Exchange, SharePoint, Teams, and OneDrive for specific content, place a legal hold (preserving content without alerting users), review it, and export it for legal proceedings or regulatory audits. DLP prevents data from being sent. Sensitivity labels classify/protect content. Insider Risk Management monitors for risky behaviour patterns. Search + hold + export for legal = eDiscovery.
Q30
A company's CISO wants to automatically detect when an employee downloads thousands of files to a USB drive in the week before their resignation, and trigger an alert for HR and legal review. Which Microsoft Purview capability is designed for this scenario?
D3
Insider Risk Management (Microsoft Purview) uses machine learning to detect patterns indicative of insider threats — such as bulk file downloads, unusual access to sensitive data, and activity spikes correlated with HR triggers like resignation notices. It generates alerts that can trigger investigation workflows. DLP blocks external sharing but doesn't detect behavioural patterns. Communication Compliance monitors message content. eDiscovery is for legal search. Behaviour + resignation trigger + alerting = Insider Risk Management.
Q31
An organisation labels a Word document as "Confidential — Internal Only" using a Microsoft Purview sensitivity label. What can the label automatically apply to the document to enforce protection?
D3
Sensitivity labels can automatically apply encryption (via Microsoft Purview Information Protection / Azure Rights Management) and access restrictions — limiting who can open, copy, print, forward, or edit the document, even if it's shared externally or emailed. Labels can also apply visual markings (watermarks, headers/footers) and control sharing settings. They do not auto-delete documents (that's retention labels) or send manager notifications. Encryption + access control = sensitivity label.
Q32
Which Microsoft tool gives a privacy team a centralised place to manage subject access requests, run privacy impact assessments, and automate data discovery workflows — specifically focused on privacy operations?
D3
Microsoft Priva is purpose-built for privacy operations: it automates Subject Rights Requests (SRRs / data subject access requests under GDPR/CCPA), helps teams assess privacy risks, monitors for personal data oversharing, and manages privacy impact assessments. Compliance Manager tracks regulatory compliance posture across frameworks. Secure Score measures security improvement. Defender for Cloud is security posture for Azure workloads. Privacy workflows + SRRs = Priva.
Q33
A compliance officer is reviewing the organisation's posture against GDPR, HIPAA, and ISO 27001. They want a single dashboard that shows which controls are implemented, calculates a compliance score, and recommends improvement actions. Which Microsoft Purview tool provides this?
D3
Microsoft Purview Compliance Manager provides a compliance score across regulatory frameworks (GDPR, HIPAA, ISO 27001, NIST, etc.), maps Microsoft-managed controls and customer-managed controls, and gives prioritised improvement actions. Don't confuse it with Secure Score — Secure Score is about security settings (MFA enabled, legacy auth blocked, etc.), while Compliance Manager is about regulatory compliance frameworks. GDPR + HIPAA + ISO dashboard = Compliance Manager.
Q34
A company creates a DLP policy in Microsoft Purview that detects when a user tries to email more than 5 credit card numbers to an external recipient, and automatically blocks the email with a policy tip shown to the sender. Which is the PRIMARY purpose of DLP in this scenario?
D3
Data Loss Prevention (DLP) policies detect sensitive information (credit card numbers, Social Security numbers, health records) in emails, Teams messages, and SharePoint files, then take protective actions — block, warn, or encrypt. The PRIMARY purpose is preventing sensitive data from leaving the organisation or being shared inappropriately. Retention labels handle archiving. Insider Risk Management handles behavioural patterns. Sensitivity labels handle encryption/access. DLP = stop data from leaking.
Q35
An IT admin needs to grant a junior helpdesk technician the ability to reset user passwords and manage licences, but NOT the ability to create or delete user accounts. Which identity management principle should guide this assignment?
D3
The principle of least privilege means giving users only the minimum permissions needed to do their job — no more. In this case, the helpdesk technician should get the "Helpdesk Administrator" or "Password Administrator" role (password resets + licence management) rather than Global Admin (which would allow account deletion). Global Admin is the most powerful role and should be reserved for very few users. Least privilege = minimum necessary access.
Q36
A Global Administrator at Woodgrove Bank needs to approve a time-limited elevation of an employee to the "Exchange Administrator" role for a 2-hour maintenance window, after which the role is automatically removed. Which Entra ID feature provides this capability?
D3
Privileged Identity Management (PIM) in Entra ID provides just-in-time (JIT) privileged access — users request elevation to a high-privilege role for a defined time window (e.g. 2 hours), requires approval and MFA, and automatically removes the role when the time expires. This reduces the "standing privilege" attack surface. PIM requires Entra ID P2 licensing. Conditional Access controls sign-in conditions. MFA verifies identity. B2B is for external partners. JIT time-limited role = PIM.
Q37
Microsoft's security model assumes that no user, device, or network should be inherently trusted — even those inside the corporate perimeter. Every access request must be verified explicitly. What is the name of this security model?
D3
Zero Trust is built on three principles: (1) Verify explicitly — always authenticate and authorise based on all available signals; (2) Use least privilege access — limit user rights with just-in-time access; (3) Assume breach — design as if attackers are already inside. Traditional perimeter security assumed that anything inside the firewall was safe. Zero Trust removes that assumption. Microsoft 365 and Azure are designed with Zero Trust principles.
Q38
A compliance team discovers that employees are routinely emailing confidential strategy documents to their personal Gmail accounts. They want a policy that warns the user and blocks the send when a document labelled "Confidential" is detected going to a non-corporate email. Which technology combination achieves this?
D3
The correct combination is Sensitivity Labels + DLP: the label classifies the document as "Confidential" and DLP policies can be configured to detect that label and block or warn when the email is addressed to an external (non-corporate) recipient. Sensitivity labels provide the classification signal; DLP provides the enforcement action. This is the standard Microsoft Purview approach to preventing labelled content from leaving the organisation.
Q39
Microsoft publishes detailed information about its data processing practices, certifications (ISO 27001, SOC 2), GDPR commitments, and audit reports so that customers can verify Microsoft's compliance posture. Where can customers find this information?
D3
The Microsoft Service Trust Portal (STP) at servicetrust.microsoft.com is where Microsoft publishes its third-party audit reports, certifications (ISO 27001, SOC 1/2, PCI DSS, FedRAMP), GDPR documentation, data protection impact assessments, and regional compliance resources. This is how customers verify Microsoft's compliance posture. Compliance Manager is for managing your organisation's compliance posture. Secure Score is for your security configuration.
Q40
A financial services company needs to monitor all Teams messages and email communications for potential violations of financial regulations (e.g., market manipulation language or offensive content). Flagged messages should be reviewed by a compliance officer before any action is taken. Which Microsoft Purview feature enables this?
D3
Communication Compliance (Microsoft Purview) monitors Teams, Exchange, and Yammer communications using trainable classifiers and keyword policies to detect regulatory violations, offensive language, or policy breaches — then routes flagged messages to designated reviewers. DLP blocks data sharing but doesn't monitor language/tone. Insider Risk Management detects file-based risky behaviour. eDiscovery is for legal search. Communication monitoring + human review = Communication Compliance.
Q41
Which Microsoft Entra ID feature allows an organisation to perform regular, automated reviews to verify that users still need access to specific applications or group memberships — and automatically remove access if reviewers don't respond or if the user no longer needs it?
D3
Access Reviews (part of Microsoft Entra ID Governance) allow organisations to schedule periodic reviews where managers or resource owners certify whether users still need access to apps, groups, or roles. If a reviewer doesn't respond or denies access, the access is automatically removed. This prevents "access creep" — accumulation of permissions over time. Requires Entra ID P2. PIM handles privileged role JIT. Conditional Access is for sign-in conditions. Access Reviews = ongoing access certification.
Q51
A company's security team is worried that employees' laptops could be used as entry points for attackers to reach internal servers. They want a Zero Trust control that specifically governs device trust and compliance before granting access to corporate resources. Which Zero Trust pillar does this concern map to?
D3
The concern is about employee laptops (devices) being exploited as entry points — this maps to the Endpoints pillar of Zero Trust. The Endpoints pillar covers all devices (PCs, mobiles, servers) that connect to corporate resources. Key M365 tools for this pillar: Microsoft Intune (device compliance), Defender for Endpoint (threat detection and AIR), and Windows Autopilot. Remember: if the scenario mentions devices or machines → Endpoints pillar.
Q52
Fabrikam's CISO is concerned about employees uploading sensitive financial documents to personal cloud storage services that IT cannot monitor. They want a Zero Trust control focused on protecting the actual content regardless of where it goes. Which Zero Trust pillar addresses this concern?
D3
The concern is about sensitive documents leaving the organisation — this maps to the Data pillar of Zero Trust. The Data pillar focuses on protecting the actual content (structured and unstructured data) wherever it lives or travels. Key M365 tools: Purview Sensitivity Labels (classify and protect the file itself with encryption), DLP (prevent sharing), and Information Protection. The key insight: if the file is encrypted with a Sensitivity Label, it remains protected even in personal cloud storage. "Protecting content wherever it goes" = Data pillar.
Q53
An organisation wants to implement Zero Trust. Their first priority is ensuring that every access request — regardless of where it originates — is authenticated using all available signals including device health, location, and sign-in risk before granting access. Which Zero Trust principle does this describe?
D3
Verify Explicitly is the Zero Trust principle that means always authenticate and authorise every request using all available signals — identity, device health, location, app sensitivity, and sign-in risk — regardless of network location. This replaces the "castle and moat" assumption that anything inside the network is safe. M365 tools that implement Verify Explicitly: MFA (verifies identity with multiple factors) and Conditional Access (evaluates every sign-in using all signals). The phrase "authenticate using all available data points" is the defining behaviour of Verify Explicitly.
Q54
Contoso invites a new partner from Woodgrove Bank to collaborate on a SharePoint project. The partner should access Contoso's SharePoint site using their existing Woodgrove Bank Microsoft 365 account without needing a new account. Which Entra feature enables this, and which identity provider works out of the box with no additional configuration?
D3
This is Entra B2B Collaboration — allowing external partners to access your tenant using their own identities. Microsoft Entra ID (work/school accounts) is one of the two default identity providers that work out of the box without any additional configuration (along with Microsoft personal accounts). Google federation and Facebook federation require admin configuration to enable. Active Directory is on-premises only and cannot directly federate to a cloud tenant as a B2B IdP. B2C is for consumer-facing applications, not partner access. Answer: B2B + Microsoft Entra ID = no extra config needed.
Q55
After a malware alert is triggered on a Windows device, Contoso's security team notices the threat was automatically investigated and the malicious file was quarantined without any manual analyst intervention. Which Microsoft Defender for Endpoint capability performed this action?
D3
Automated Investigation and Remediation (AIR) is the Defender for Endpoint capability that automatically investigates alerts and takes remediation actions (like quarantining files or terminating processes) without manual analyst intervention. AIR mimics what a security analyst would do — inspecting process trees, file behaviour, and network connections — then acts automatically. ASR blocks risky behaviours before they execute. Network Protection blocks outbound connections. Secure Score measures posture but takes no direct remediation action. "Automatically investigated and remediated" = AIR.
Q56
An organisation needs to ensure that all Teams chat messages and Exchange emails are automatically preserved for 5 years to meet industry regulations, and then deleted after that period. They do not need immutable records — just automated lifecycle management. Which Purview feature should they use?
D3
Retention Policies are the right tool here — they automatically define how long content is kept and when it is deleted across Exchange, Teams, SharePoint, and OneDrive. This is broad, automated lifecycle management at scale. eDiscovery Holds preserve content for an active legal case but are reactive and case-specific — not automated lifecycle management. Records Management is for content that must be immutable (cannot be edited or deleted) — the question says they don't need immutability. DLP prevents sharing, not lifecycle management. "5-year keep then delete, automated" = Retention Policies.
Q63
A compliance officer at a financial services firm wants written assurance that Microsoft does not scan or mine employee emails, Teams messages, or files to serve targeted advertising to Microsoft's own business. Which of Microsoft's privacy principles directly addresses this guarantee?
D3
No Content-Based Targeting is the Microsoft privacy principle that specifically states Microsoft does not scan, mine, or use the content of customers' emails, Teams messages, documents, or other data to serve targeted advertisements. This is distinct from the "Control" principle (which covers data ownership), "Transparency" (which covers disclosure about data collection practices), and "Security" (which covers how data is protected). The question's key phrase is "scan or mine... to serve targeted advertising" — that maps directly to No Content-Based Targeting.
Q67
A company wants to replace their VPN solution so that remote employees can access internal line-of-business apps without gaining access to the entire corporate network. Which Microsoft product provides this capability using Zero Trust principles?
D3
Microsoft Entra Private Access (EPA) is Microsoft's Zero Trust Network Access (ZTNA) solution designed to replace VPNs. Unlike VPNs — which grant full network access once connected — EPA grants per-application access only, enforced by Conditional Access, MFA, and identity-based policies. Internal apps are never exposed to the open internet. EPA is the private-app component of Microsoft Global Secure Access (GSA). Entra Internet Access (A) handles internet/SaaS traffic, not internal app access.
Q68
Which two services together form Microsoft Global Secure Access (GSA)?
D3
Microsoft Global Secure Access (GSA) is the unified Security Service Edge (SSE) umbrella that combines exactly two services: Microsoft Entra Internet Access (IEA) — the cloud Secure Web Gateway protecting internet and SaaS traffic — and Microsoft Entra Private Access (EPA) — the Zero Trust Network Access solution replacing VPN for private/internal app access. Together they secure access to internet, SaaS, and private apps using identity, Conditional Access, and Microsoft's global private network.
Q69
